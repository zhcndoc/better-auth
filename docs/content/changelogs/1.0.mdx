版本更新

Better Auth 带有内置的 CLI，帮助你管理核心功能和插件所需的数据库 schema。

## 生成

`generate` 命令创建 Better Auth 所需的 schema。如果你使用的是 Prisma 或 Drizzle 等数据库适配器，此命令会为你的 ORM 生成合适的 schema。如果你使用内置的 Kysely 适配器，它会生成一个可以直接在数据库上运行的 SQL 文件。

```bash title="Terminal"
npx auth@latest generate
```

### 选项

- `--output` - 保存生成 schema 的位置。对于 Prisma，会保存在 prisma/schema.prisma。对于 Drizzle，会保存在项目根目录的 schema.ts。对于 Kysely，是保存为项目根目录的 schema.sql 的 SQL 文件。
- `--config` - 指定 Better Auth 配置文件的路径。默认情况下，CLI 会在 **./**、**./utils**、**./lib**，或 `src` 目录下的这些文件夹内搜索 better-auth.ts 文件。
- `--y` - 跳过确认提示，直接生成 schema。

## 迁移

`migrate` 命令将 Better Auth 的 schema 直接应用到你的数据库。如果你使用的是内置的 Kysely 适配器，可以使用此命令。

```bash title="Terminal"
npx auth@latest migrate
```

### 选项

- `--config` - 指定 Better Auth 配置文件的路径。默认情况下，CLI 会在 **./**、**./utils**、**./lib**，或 `src` 目录下的这些文件夹内搜索 better-auth.ts 文件。
- `--y` - 跳过确认提示，直接应用 schema。

## 常见问题

**错误：找不到模块 X**

如果出现此错误，说明 CLI 无法解析 Better Auth 配置文件中的导入模块。我们正在修复许多这类问题，但目前你可以尝试：

- 移除配置文件中的任何导入别名，改用相对路径。运行 CLI 后，你可以再恢复使用别名。

## Secret

CLI 还提供了为你的 Better Auth 实例生成密钥的方法。

```bash title="Terminal"
npx auth@latest secret
```
