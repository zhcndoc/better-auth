在本指南中，我们将带你一步步创建你的第一个 Better Auth 插件。


<Callout type="warn">
本指南假设你已经完成了 <Link href="/docs/installation">Better Auth 的基础设置</Link>，并准备好创建你的第一个插件。
</Callout>

<Steps>
<Step>
## 规划你的想法
开始之前，你必须知道你打算创建什么插件。

在本指南中，我们将创建一个 **生日插件** 来记录用户的生日。
</Step>

<Step>
## 先创建服务器插件
Better Auth 插件是成对工作的：一个 <Link href="/docs/concepts/plugins#create-a-server-plugin">服务器插件</Link> 和一个 <Link href="/docs/concepts/plugins#creating-a-client-plugin">客户端插件</Link>。
服务器插件构成你的认证系统的基础，而客户端插件则提供方便的前端 API 来与服务器实现交互。

<Callout>
你可以在我们的 <Link href="/docs/concepts/plugins#creating-a-plugin">文档</Link> 中阅读更多关于服务器/客户端插件的内容。
</Callout>

### 创建服务器插件
找到一个合适的位置创建你的生日插件文件夹，并在其中创建一个 `index.ts` 文件。
<Files>
  <Folder name="birthday-plugin" defaultOpen>
    <File name="index.ts" />
  </Folder>
</Files>
在 `index.ts` 文件中，我们将导出一个函数，代表我们的服务器插件。
稍后我们会将它添加到 `auth.ts` 文件中的插件列表里。

```ts title="index.ts"
import { createAuthClient } from "better-auth/client";
import type { BetterAuthPlugin } from "better-auth";

export const birthdayPlugin = () =>
  ({
    id: "birthdayPlugin",
  } satisfies BetterAuthPlugin);

```
虽然这段代码目前并没有实际功能，但你已经技术上完成了你的第一个插件，恭喜！🎉

</Step>

<Step>
### 定义 schema
为了保存每个用户的生日数据，我们必须在 `user` 模型上创建 schema。

在这里创建 schema，也允许 <Link href="/docs/concepts/cli">Better Auth 的 CLI</Link> 自动生成更新数据库所需的 schema。

<Callout type="info">
你可以在 <Link href="/docs/concepts/plugins#schema">这里了解更多关于插件 schema</Link> 的内容。
</Callout>

```ts title="index.ts"
//...
export const birthdayPlugin = () =>
  ({
    id: "birthdayPlugin",
    schema: {// [!code highlight]
      user: {// [!code highlight]
        fields: {// [!code highlight]
          birthday: {// [!code highlight]
            type: "date", // 字符串、数字、布尔值、日期 // [!code highlight]
            required: true, // 新记录时此字段是否必填（默认：false） // [!code highlight]
            unique: false, // 此字段是否唯一（默认：false） // [!code highlight]
          },// [!code highlight]
        },// [!code highlight]
      },// [!code highlight]
    },
  } satisfies BetterAuthPlugin);
```

</Step>

<Step>
### 授权逻辑
在本示例中，我们将设置认证逻辑，检查并确保注册的用户年龄大于 5 岁。
同样的概念也可用于验证用户同意服务条款（TOS）等场景。

为此，我们将使用 <Link href="/docs/concepts/plugins#hooks">Hooks</Link>，它允许我们在操作执行前或执行后运行代码。

```ts title="index.ts"
export const birthdayPlugin = () => ({
    //...
    // 在我们的例子中，我们要写授权逻辑，
    // 也就是说，我们希望在操作执行"之前"拦截。
    hooks: {
      before: [
        {
          matcher: (context) => /* ... */,
          handler: createAuthMiddleware(async (ctx) => {
            //...
          }),
        },
      ],
    },
} satisfies BetterAuthPlugin)
```

这里我们想匹配所有访问注册路径的请求：
```ts title="Before hook"
{
  matcher: (context) => context.path.startsWith("/sign-up/email"),
  //...
}
```

接着针对逻辑，我们写以下代码检查用户生日是否满足年龄大于 5 岁：
```ts title="Imports"
import { createAuthMiddleware, APIError } from "better-auth/api";
```
```ts title="Before hook"
{
  //...
  handler: createAuthMiddleware(async (ctx) => {
    const { birthday } = ctx.body;
    if(!(birthday instanceof Date)) {
      throw new APIError("BAD_REQUEST", { message: "生日必须是 Date 类型。" });
    }

    const today = new Date();
    const fiveYearsAgo = new Date(today.setFullYear(today.getFullYear() - 5));

    if(birthday >= fiveYearsAgo) {
      throw new APIError("BAD_REQUEST", { message: "用户必须年满 5 岁以上。" });
    }

    return { context: ctx };
  }),
}
```

**授权通过！** 🔒

我们现在成功编写了确保用户年龄大于 5 岁的授权逻辑！

</Step>

<Step>
## 客户端插件
我们已经接近完成了！🏁

现在服务器插件已经创建完成，下一步是开发客户端插件。
由于这个插件没有太多前端 API，所以工作量不大！

首先，创建 `client.ts` 文件：
<Files>
  <Folder name="birthday-plugin" defaultOpen>
    <File name="index.ts" />
    <File name="client.ts" />
  </Folder>
</Files>
然后，添加以下代码：
```ts title="client.ts"
import type { BetterAuthClientPlugin } from "better-auth/client";
import type { birthdayPlugin } from "./index"; // 确保以类型方式导入服务器插件 // [!code highlight]

type BirthdayPlugin = typeof birthdayPlugin;

export const birthdayClientPlugin = () => {
  return {
    id: "birthdayPlugin",
    $InferServerPlugin: {} as ReturnType<BirthdayPlugin>,
  } satisfies BetterAuthClientPlugin;
};
```
我们做的是让客户端插件推断服务器插件中 schema 定义的类型。

就是这样！这就是生日客户端插件的全部内容。🎂

</Step>

<Step>
## 启动你的插件！
客户端和服务器插件都已准备好，最后一步是分别将它们导入到 `auth-client.ts` 和 `server.ts` 中以启动插件。

### 服务器启动
```ts title="server.ts"
import { betterAuth } from "better-auth";
import { birthdayPlugin } from "./birthday-plugin";// [!code highlight]
 
export const auth = betterAuth({
    plugins: [
      birthdayPlugin(),// [!code highlight]
    ]
});
```

### 客户端启动
```ts title="auth-client.ts"
import { createAuthClient } from "better-auth/client";
import { birthdayClientPlugin } from "./birthday-plugin/client";// [!code highlight]
 
const authClient = createAuthClient({
    plugins: [
      birthdayClientPlugin()// [!code highlight]
    ]
});
```

### 哦对，还有 schemas！
别忘了将 `birthday` 字段添加到你的 `user` 表模型中！

或者，使用 `generate` <Link href="/docs/concepts/cli#generate">CLI 命令</Link>：
```bash
npx auth@latest generate
```

</Step>
