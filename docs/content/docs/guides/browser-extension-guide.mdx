---
title: 浏览器扩展指南
description: 使用 Better Auth 创建浏览器扩展的逐步指南。
---

在本指南中，我们将带您了解如何使用 <Link href="https://docs.plasmo.com/">Plasmo</Link> 结合 Better Auth 进行身份验证来创建浏览器扩展。

如果您想查看完整示例，可以参考 <Link href="https://github.com/better-auth/examples/tree/main/browser-extension-example">浏览器扩展示例</Link>。

<Callout type="warn">
  Plasmo 框架不提供浏览器扩展的后端支持。
  本指南假设您已经{" "}
  <Link href="/docs/integrations/hono">设置好了 Better Auth 后端</Link>，
  并准备创建一个浏览器扩展来连接它。
</Callout>

<Steps>

    <Step>
        ## 设置与安装

        使用 TailwindCSS 和 src 目录初始化一个新的 Plasmo 项目。

        ```bash
        pnpm create plasmo --with-tailwindcss --with-src
        ```

        然后，安装 Better Auth 包。

        ```bash
        pnpm add better-auth
        ```

        启动 Plasmo 开发服务器，运行以下命令。

        ```bash
        pnpm dev
        ```
    </Step>


    <Step>
        ## 配置 tsconfig

        配置 `tsconfig.json` 文件以启用 `strict` 模式。

        本演示中，我们还将导入路径别名从 `~` 改为 `@` 并指向 `src` 目录。

        ```json title="tsconfig.json"
        {
            "compilerOptions": {
                "paths": {
                    "@/_": [
                        "./src/_"
                    ]
                },
                "strict": true,
                "baseUrl": "."
            }
        }
        ```
    </Step>


    <Step>
        ## 创建客户端认证实例

        在 `src/auth/auth-client.ts` 新建一个文件并添加以下代码。

       <Files>
            <Folder name="src" defaultOpen>
                <Folder name="auth" defaultOpen>
                    <File name="auth-client.ts" />
                </Folder>
            </Folder>
       </Files>

        ```ts title="auth-client.ts"
        import { createAuthClient } from "better-auth/react"

        export const authClient = createAuthClient({
            baseURL: "http://localhost:3000" /* 你的 Better Auth 后端基础 URL。 */,
            plugins: [],
        });
        ```
    </Step>

    <Step>
        ## 配置 manifest

        我们必须确保扩展知道 Better Auth 后端的 URL。

        打开你的 package.json 文件，添加以下内容。

        ```json title="package.json"
        {
            //...
            "manifest": {
                "host_permissions": [
                    "https://URL_TO_YOUR_BACKEND" // 本地地址同样可用（例如 http://localhost:3000）
                ]
            }
        }
        ```
    </Step>


    <Step>
        ## 你现在准备好了！

        你已经为浏览器扩展设置好了 Better Auth。

        添加你想要的 UI，创造你的理想扩展！

        想了解更多客户端 Better Auth API，请查看 <Link href="/docs/concepts/client">客户端文档</Link>。

        这里有个快速示例 😎

        ```tsx title="src/popup.tsx"
        import { authClient } from "./auth/auth-client"


        function IndexPopup() {
            const {data, isPending, error} = authClient.useSession();
            if(isPending){
                return <>加载中...</>
            }
            if(error){
                return <>错误: {error.message}</>
            }
            if(data){
                return <>已登录为 {data.user.name}</>
            }
        }

        export default IndexPopup;
        ```

    </Step>


    <Step>
        ## 打包你的扩展

        若要获得生产构建，运行以下命令。

        ```bash
        pnpm build
        ```

        访问 <Link href="chrome://extensions" target="_blank">chrome://extensions</Link> 并启用开发者模式。

        <img src="https://docs.plasmo.com/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Fdeveloper_mode.76f090f7.png&w=1920&q=75" />

        点击“加载已解压的扩展程序”，导航至你的扩展的 `build/chrome-mv3-dev`（或 `build/chrome-mv3-prod`）目录。

        若要查看你的弹出窗口，点击 Chrome 工具栏的拼图图标，然后点击你的扩展。

        了解更多关于 <Link href="https://docs.plasmo.com/framework#loading-the-extension-in-chrome">打包扩展</Link> 的内容。
    </Step>

    <Step>
        ## 配置服务器认证实例

        首先，我们需要你的扩展 URL。
        
        扩展 URL 格式如下：`chrome-extension://YOUR_EXTENSION_ID`。

        你可以在 <Link href="chrome://extensions" target="_blank">chrome://extensions</Link> 找到你的扩展 ID。

        <img src="/extension-id.png" width={500} />

        打开服务器的 auth 文件，确保你的扩展 URL 已添加到 `trustedOrigins` 列表中。


        ```ts title="server.ts"
        import { betterAuth } from "better-auth"
        import { auth } from "@/auth/auth"

        export const auth = betterAuth({
            trustedOrigins: ["chrome-extension://YOUR_EXTENSION_ID"],
        })
        ```

        如果你开发多个扩展或需要支持不同 ID 的浏览器扩展，可以使用通配符模式：

        ```ts title="server.ts"
        export const auth = betterAuth({
            trustedOrigins: [
                // 支持特定扩展 ID
                "chrome-extension://YOUR_EXTENSION_ID",
                
                // 或使用通配符支持多个扩展（安全性较低）
                "chrome-extension://*"
            ],
        })
        ```

        <Callout type="warn">
          使用通配符（例如 `chrome-extension://*`）会降低安全性，因为它信任所有扩展。 
          更安全的做法是明确列出你信任的每个扩展 ID。仅在开发和测试时使用通配符。
        </Callout>
    </Step>

    <Step>
        ## 就这样！

        所有设置完成！你现在可以开始开发你的扩展了。 🎉
    </Step>

</Steps>
