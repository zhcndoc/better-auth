邮件是 Better Auth 的关键部分，所有用户无论使用何种认证方式都必须使用邮件。Better Auth 开箱即用提供邮件和密码认证，并提供了很多工具帮助你管理邮件验证、密码重置等功能。

## 邮件验证

### 在你的应用中添加邮件验证

要启用邮件验证，你需要传入一个发送验证邮件的函数，该邮件带有一个链接。

- **sendVerificationEmail**：当邮件验证启动时触发的函数。它接收一个包含以下属性的数据对象：
  - `user`：包含邮件地址的用户对象。
  - `url`：用户必须点击以验证邮件的验证链接。
  - `token`：验证令牌，用于完成邮件验证，可用于自定义验证链接的实现。

* **sendVerificationEmail**: This function is triggered when email verification starts. It accepts a data object with the following properties:
  * `user`: The user object containing the email address.
  * `url`: The verification URL the user must click to verify their email.
  * `token`: The verification token used to complete the email verification to be used when implementing a custom verification URL.

and a `request` object as the second parameter.

```ts title="auth.ts"
import { betterAuth } from 'better-auth';
import { sendEmail } from './email'; // 你的邮件发送函数

export const auth = betterAuth({
    emailVerification: {
        sendVerificationEmail: async ({ user, url, token }, request) => {
            void sendEmail({
                to: user.email,
                subject: '验证你的邮件地址',
                text: `点击链接验证你的邮件：${url}`
            })
        }
    }
})
```

<Callout type="warn">
  Avoid awaiting the email sending to prevent
  timing attacks. On serverless platforms, use `waitUntil` or similar to ensure the email is sent.
</Callout>

### 触发邮件验证

你可以通过多种方式触发邮件验证：

#### 1. 注册时

To automatically send a verification email at signup, set `emailVerification.sendOnSignUp` to `true`.

```ts title="auth.ts"
import { betterAuth } from 'better-auth';

export const auth = betterAuth({
    emailVerification: {
        sendOnSignUp: true
    }
})
```

这样用户注册时会发送验证邮件。对于社交登录，邮件验证状态会从 SSO 读取。

<Callout>
  With `sendOnSignUp` enabled, when the user logs in with an SSO that does not claim the email as verified, Better Auth will dispatch a verification email, but the verification is not required to login even when `requireEmailVerification` is enabled.
</Callout>

#### 2. 强制邮件验证

如果启用强制邮件验证，用户必须验证邮件后才能登录。每次用户尝试登录时，都会调用 `sendVerificationEmail`。

<Callout>
  This only works if you have `sendVerificationEmail` implemented, if `sendOnSignIn` is set to true and if the user is trying to sign in with email and password.
</Callout>

```ts title="auth.ts"
import { betterAuth } from "better-auth";
import { sendEmail } from './email'; // 你的邮件发送函数

export const auth = betterAuth({
  emailVerification: {
    sendVerificationEmail: async ({ user, url }) => {
      void sendEmail({
        to: user.email,
        subject: "验证你的邮件地址",
        text: `点击链接验证你的邮件：${url}`,
      });
    },
    sendOnSignIn: true,
  },
  emailAndPassword: {
    requireEmailVerification: true,
  },
});
```

如果用户尝试在未验证邮件的情况下登录，你可以捕获错误并向用户显示提示。

```ts title="auth-client.ts"
await authClient.signIn.email({
    email: "email@example.com",
    password: "password"
}, {
    onError: (ctx) => {
        // 处理错误
        if(ctx.error.status === 403) {
            alert("请验证你的邮件地址")
        }
        // 你也可以显示原始错误消息
        alert(ctx.error.message)
    }
})
```

#### 3. 手动触发

你也可以通过调用 `sendVerificationEmail` 手动触发邮件验证。

```ts
await authClient.sendVerificationEmail({
    email: "user@email.com",
    callbackURL: "/" // 验证后重定向的 URL
})
```

### 验证邮件

如果用户点击验证链接，他们的邮件会被自动验证，并被重定向至 `callbackURL`。

对于手动验证，你可以发送带有 `token` 的自定义链接给用户，然后调用 `verifyEmail` 方法。

```ts
await authClient.verifyEmail({
    query: {
        token: "" // 在这里传递令牌
    }
})
```

### 验证后自动登录

若你希望用户成功验证邮件后自动登录，请将 `autoSignInAfterVerification` 选项设置为 `true`：

```ts
import { betterAuth } from "better-auth";

const auth = betterAuth({
    //... 其他配置
    emailVerification: {
        autoSignInAfterVerification: true
    }
})
```

### 验证前回调

你可以使用 `beforeEmailVerification` 回调，在用户邮件被标记为已验证之前运行自定义代码。这对于验证、预检查或在最终确定验证之前准备数据非常有用。

```ts title="auth.ts"
import { betterAuth } from 'better-auth';

export const auth = betterAuth({
    emailVerification: {
        async beforeEmailVerification(user, request) {
            // 运行验证前逻辑
            console.log(`About to verify ${user.email}`);
        }
    }
})
```

### 验证成功后回调

你可以使用 `afterEmailVerification` 回调，在用户成功验证邮件后立即执行自定义代码。这对于触发副作用非常有用，比如授予特殊权限或记录日志。

`afterEmailVerification` 函数会在用户邮件确认时自动调用，接收 `user` 对象和 `request` 详情，以便你针对特定用户执行操作。

示例如下：

```ts title="auth.ts"
import { betterAuth } from 'better-auth';

export const auth = betterAuth({
    emailVerification: {
        async afterEmailVerification(user, request) {
            // 你的自定义逻辑，例如授予高级功能访问权限
            console.log(`${user.email} 已成功验证！`);
        }
    }
})
```

### 重复注册尝试回调

当有人尝试使用已经存在的邮箱注册时，会触发 `onExistingUserSignUp` 回调。这对于通知现有用户有人尝试使用他们的邮箱地址注册非常有用。

<Callout>
  This callback is only called when `emailAndPassword.requireEmailVerification` is set to `true`, which enables the generic duplicate response (synthetic user) instead of throwing a duplicate-email error.
</Callout>

```ts title="auth.ts"
import { betterAuth } from 'better-auth';

export const auth = betterAuth({
    emailAndPassword: {
        enabled: true,
        onExistingUserSignUp: async ({ user }, request) => {
            // 通知现有用户有人尝试使用他们的邮箱注册
            console.log(`Someone tried to sign up with ${user.email}`);
        }
    }
})
```

## 密码重置邮件

忘记密码时，重置密码功能允许用户重设密码。Better Auth 提供了简单的方式实现密码重置。

你可以通过传入发送密码重置邮件的函数来启用密码重置功能，该邮件带有一个链接。

```ts title="auth.ts"
import { betterAuth } from 'better-auth';
import { sendEmail } from './email'; // 你的邮件发送函数

export const auth = betterAuth({
    emailAndPassword: {
        enabled: true,
        sendResetPassword: async ({ user, url, token }, request) => {
            void sendEmail({
                to: user.email,
                subject: '重置你的密码',
                text: `点击链接重置你的密码：${url}`
            })
        }
    }
})
```

<Callout type="warn">
  Avoid awaiting the email sending to prevent
  timing attacks. On serverless platforms, use `waitUntil` or similar to ensure the email is sent.
</Callout>

更多关于如何在应用中实现密码重置的信息，请查看 [邮件和密码](/docs/authentication/email-password#request-password-reset) 指南。
你也可以参阅 [OTP 验证](/docs/plugins/email-otp#reset-password-with-otp) 指南，了解如何用 OTP 实现密码重置。
