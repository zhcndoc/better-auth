Better Auth 托管了一个**远程 MCP 服务器**，可为任何支持 MCP 的客户端（Cursor、Claude Code、Open Code 等）提供文档搜索、示例和设置帮助。

**端点：** `https://mcp.better-auth.com/mcp`

这与 [MCP 插件](/docs/plugins/mcp) 是分开的，后者会为你的应用添加 MCP *提供者认证*。上面的服务器用于在你的编辑器或代理中**消费** Better Auth 文档。

## 使用 CLI

运行 [Better Auth CLI](/docs/concepts/cli) 并选择你的客户端：

```bash title="terminal"
npx auth@latest mcp
```

在不带标志的情况下，该命令会列出支持的目标。你也可以直接传入一个目标：

<Tabs items={["Cursor", "Claude Code", "Open Code", "Manual"]}>
  <Tab value="Cursor">
    ```bash title="terminal"
    npx auth@latest mcp --cursor
    ```

    这会在 Cursor 中打开一个深度链接，以便将服务器添加到你的 MCP 配置中。

    你也可以使用此网站上的单击控件：

    <AddToCursor />
  </Tab>

  <Tab value="Claude Code">
    ```bash title="terminal"
    npx auth@latest mcp --claude-code
    ```

    这会使用 Better Auth URL 运行 `claude mcp add`。如果失败，CLI 会打印出确切的命令供你手动运行。
  </Tab>

  <Tab value="Open Code">
    ```bash title="terminal"
    npx auth@latest mcp --open-code
    ```

    这会在当前目录的 `opencode.json` 中合并一个 `better-auth` 条目（远程传输，使用上方的 URL）。
  </Tab>

  <Tab value="Manual">
    ```bash title="terminal"
    npx auth@latest mcp --manual
    ```

    在当前工作目录中写入或合并 `mcp.json`，内容为：

    ```json
    {
      "better-auth": {
        "url": "https://mcp.better-auth.com/mcp"
      }
    }
    ```

    对于 **Cursor**，你可以将相同的 `better-auth` 条目添加到 MCP 配置中——通常为 `~/.cursor/mcp.json`（用户）或 `.cursor/mcp.json`（项目），使用你的 Cursor 版本所需的 HTTP 服务器格式（`url` 表示远程 MCP）。

    在更改后，请重启或重新加载客户端中的 MCP。
  </Tab>
</Tabs>
