## 安装

```tsx
import { betterAuth } from "better-auth";
import { sentinel } from "@better-auth/infra";

export const auth = betterAuth({
  plugins: [
    sentinel({
      security: {
        // 在此配置安全功能
      },
    }),
  ],
});
```

## 配置选项

### SentinelOptions

| 选项 | 类型 | 说明 |
|--------|------|-------------|
| `apiUrl` | `string` | Better Auth基础设施 API URL |
| `kvUrl` | `string` | 用于限速数据的KV存储URL |
| `apiKey` | `string` | 用于认证的API密钥 |
| `security` | `SecurityOptions` | 安全功能配置 |

### SecurityOptions

```ts
interface SecurityOptions {
  unknownDeviceNotification?: boolean;
  credentialStuffing?: CredentialStuffingConfig;
  impossibleTravel?: ImpossibleTravelConfig;
  geoBlocking?: GeoBlockingConfig;
  botBlocking?: boolean | { action: SecurityAction };
  suspiciousIpBlocking?: boolean | { action: SecurityAction };
  velocity?: VelocityConfig;
  freeTrialAbuse?: FreeTrialAbuseConfig;
  compromisedPassword?: CompromisedPasswordConfig;
  emailValidation?: EmailValidationConfig;
  staleUsers?: StaleUsersConfig;
  challengeDifficulty?: number;
}

type SecurityAction = "log" | "challenge" | "block";
```

## 安全功能

### 凭证填充防护

通过跟踪每个访客的登录失败次数，检测并阻止凭证填充攻击。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    credentialStuffing: {
      enabled: true,
      thresholds: {
        challenge: 3,  // 失败3次后发起PoW挑战
        block: 5,      // 失败5次后封禁
      },
      windowSeconds: 3600,     // 1小时时间窗口
      cooldownSeconds: 900,    // 封禁后15分钟冷却期
    },
  },
}),
```

**工作原理：**
1. 跟踪每个访客ID的登录失败次数
2. 达到挑战阈值时，发起工作量证明（PoW）挑战
3. 达到封禁阈值时，完全封禁该访客
4. 登录成功时自动清除失败次数

### 不可能的旅行检测

检测用户在极短时间内从地理上相距遥远的地方登录。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    impossibleTravel: {
      enabled: true,
      maxSpeedKmh: 1000,    // 最大合理旅行速度
      action: "challenge",  // 可选："log"、"challenge"或"block"
    },
  },
}),
```

**示例：** 如果用户先后在纽约和30分钟后在东京登录，这被视为不可能的旅行（即需要超过1000公里/小时的速度）。

### 免费试用滥用防护

通过设备指纹防止用户创建多个账户以滥用免费试用。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    freeTrialAbuse: {
      enabled: true,
      thresholds: {
        challenge: 2,
        block: 3,
      },
      maxAccountsPerVisitor: 3,
      action: "block",
    },
  },
}),
```

**工作原理：**
1. 跟踪每个访客指纹创建的账户数
2. 超过阈值时阻止新账户的创建
3. 有效防止免费套餐滥用

### 已泄露密码检测

通过查询 HaveIBeenPwned 数据库检查密码，检测泄露凭证。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    compromisedPassword: {
      enabled: true,
      action: "block",      // 可选："log"、"challenge"或"block"
      minBreachCount: 1,    // 触发的最小泄露次数
    },
  },
}),
```

**隐私说明：** 采用k-匿名技术 — 仅将密码哈希的前5个字符发送到API，完整密码绝不发送。

### 过期账户监控

检测沉睡账户突然活跃，可能表示账户被接管。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    staleUsers: {
      enabled: true,
      staleDays: 90,           // 账户超过90天视为沉睡
      action: "log",           // 可选："log"、"challenge"或"block"
      notifyUser: true,        // 向用户发送邮件通知
      notifyAdmin: true,       // 向管理员发送邮件通知
      adminEmail: "admin@yourapp.com",
    },
  },
}),
```

**通知内容包括：**
- 登录时间和地点
- 距离上次活动的天数
- 设备信息

### 地理封禁

封禁或挑战特定国家的用户。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    geoBlocking: {
      allowList: ["US", "CA", "GB"],  // 仅允许这些国家
      // 或者
      denyList: ["XX", "YY"],          // 封禁这些国家
      action: "block",                  // 可选："challenge"或"block"
    },
  },
}),
```

使用 ISO 3166-1 alpha-2 国家代码。

### 机器人封禁

检测并阻止自动化机器流量。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    botBlocking: true,
    // 或使用自定义操作
    botBlocking: {
      action: "challenge",  // 可选："log"、"challenge"或"block"
    },
  },
}),
```

### 可疑IP检测

封禁来自已知恶意IP地址的请求。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    suspiciousIpBlocking: true,
    // 或使用自定义操作
    suspiciousIpBlocking: {
      action: "block",
    },
  },
}),
```

### 访问速率限制 / 频率控制

限制各种操作的请求速率。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    velocity: {
      enabled: true,
      thresholds: {
        challenge: 10,
        block: 20,
      },
      maxSignupsPerVisitor: 5,
      maxPasswordResetsPerIp: 10,
      maxSignInsPerIp: 50,
      windowSeconds: 3600,
      action: "challenge",
    },
  },
}),
```

### 邮箱验证

封禁一次性邮箱地址并验证邮箱域名。

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    emailValidation: {
      enabled: true,
      strictness: "medium",  // 可选："low"、"medium"、"high"
      action: "block",
    },
  },
}),
```

**严格级别：**
- `low` - 仅封禁已知一次性邮箱域名
- `medium` - 还会检查有效 MX 记录
- `high` - 追加启发式检查

## 工作量证明（PoW）挑战

当安全检查导致“challenge”动作时，Sentinel会发起工作量证明（PoW）挑战，客户端必须完成挑战。

### PoW 工作原理

1. 服务器发出加密挑战
2. 客户端必须找到满足难度要求的解答
3. 解答计算开销大，但验证快速
4. 有效阻止自动化攻击，允许合法用户通过

### 挑战难度

```ts
sentinel({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  security: {
    challengeDifficulty: 18,  // 默认难度等级
  },
}),
```

难度越高 = 计算量越大 = 攻击者越慢。

## 客户端集成

### sentinelClient()

客户端插件负责设备指纹识别及自动完成PoW挑战。

```ts
import { createAuthClient } from "better-auth/client";
import { sentinelClient } from "@better-auth/infra/client";

export const authClient = createAuthClient({
  plugins: [
    sentinelClient({
      autoSolveChallenge: true,
    }),
  ],
});
```

### 配置项

| 选项 | 类型 | 默认值 | 说明 |
|--------|------|---------|-------------|
| `autoSolveChallenge` | `boolean` | `true` | 自动解决PoW挑战 |

### 浏览器指纹

客户端通过 `X-Visitor-ID` 请求头自动包含访客ID指纹，用于：

- 凭证填充检测
- 免费试用滥用防护
- 设备追踪

### PoW解答请求头

启用自动解决后，解答通过 `X-PoW-Solution` 头发送。

## 安全事件

Sentinel跟踪以下安全事件类型：

| 事件类型 | 说明 |
|------------|-------------|
| `security_blocked` | 请求被封禁 |
| `security_allowed` | 挑战通过后请求被允许 |
| `security_credential_stuffing` | 检测到凭证填充 |
| `security_impossible_travel` | 检测到不可能的旅行 |
| `security_geo_blocked` | 地理封禁触发 |
| `security_bot_blocked` | 机器人被检测并封禁 |
| `security_suspicious_ip` | 检测到异常IP |
| `security_velocity_exceeded` | 超出速率限制 |
| `security_free_trial_abuse` | 检测到免费试用滥用 |
| `security_compromised_password` | 检测到泄露密码 |
| `security_stale_account` | 沉睡账户重新激活 |

这些事件可在安全控制台查看，并包含在审计日志中。

## 完整示例

```ts
import { betterAuth } from "better-auth";
import { sentinel } from "@better-auth/infra";

export const auth = betterAuth({
  plugins: [
    sentinel({
      apiKey: process.env.BETTER_AUTH_API_KEY,
      security: {
        // 核心防护
        credentialStuffing: {
          enabled: true,
          thresholds: { challenge: 3, block: 5 },
        },
        compromisedPassword: {
          enabled: true,
          action: "block",
        },
        emailValidation: {
          enabled: true,
          strictness: "medium",
          action: "block",
        },
        
        // 基于位置的策略
        impossibleTravel: {
          enabled: true,
          action: "challenge",
        },
        geoBlocking: {
          denyList: ["XX"],
          action: "block",
        },
        
        // 滥用防护
        freeTrialAbuse: {
          enabled: true,
          maxAccountsPerVisitor: 3,
          action: "block",
        },
        velocity: {
          enabled: true,
          maxSignupsPerVisitor: 5,
          action: "challenge",
        },
        
        // 机器人防护
        botBlocking: { action: "challenge" },
        suspiciousIpBlocking: { action: "block" },
        
        // 账户监控
        staleUsers: {
          enabled: true,
          staleDays: 90,
          notifyUser: true,
          notifyAdmin: true,
          adminEmail: "security@yourapp.com",
        },
      },
    }),
  ],
});
```

## 最佳实践

1. **先从日志开始** - 初期将动作设置为“log”，以了解流量模式，再决定是否封禁。

2. **调整阈值** - 每个应用都不同，监控误报率并相应调整阈值。

3. **先使用挑战，后使用封禁** - 挑战能阻挡自动化攻击，同时允许合法用户访问。

4. **启用客户端自动解答** - 确保配置了 `sentinelClient({ autoSolveChallenge: true })` ，以优化用户体验。

5. **监控安全事件** - 定期查看安全控制台，识别攻击趋势。

6. **设置管理员通知** - 对关键事件如沉睡账户激活启用 `notifyAdmin`。

## 疑难解答

### 缺少API密钥警告

```
[Sentinel] Missing BETTER_AUTH_API_KEY. Security checks may fall back to allow mode.
```

请确保正确设置了API密钥环境变量。

### 挑战无法执行

如果PoW挑战无法自动完成：
1. 确认客户端已安装 `sentinelClient()` 插件
2. 检查 `autoSolveChallenge` 是否为 `true`
3. 确认客户端能够访问服务器

### 误报率过高

如果误伤合法用户：
1. 提高阈值
2. 将动作从“block”改为“challenge”
3. 查看安全事件，找出误报原因
