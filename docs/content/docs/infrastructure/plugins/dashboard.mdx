仪表盘插件是您的 Better Auth 实例和 Better Auth 基础设施之间的核心连接。它为网页仪表盘提供实时数据，跟踪用户活动，并启用管理员 API。

## 安装

```ts
import { betterAuth } from "better-auth";
import { dash } from "@better-auth/infra";

export const auth = betterAuth({
  plugins: [
    dash(),
  ],
});
```

## 配置选项

| 选项             | 类型     | 描述                        |
| ------------------ | -------- | ------------------------------ |
| `apiUrl`           | `string` | Better Auth 基础设施 API URL |
| `kvUrl`            | `string` | 用于缓存的 KV 存储 URL           |
| `apiKey`           | `string` | 您的身份验证 API 密钥            |
| `activityTracking` | `object` | 活动跟踪配置                    |

## 活动跟踪

跟踪用户在您的应用中的最后活跃时间。启用后，用户活动时会自动更新 `lastActiveAt` 字段。

```ts
dash({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  activityTracking: {
    enabled: true,
    updateInterval: 300000,  // 更新间隔，单位毫秒（默认：5 分钟）
  },
}),
```

### 模式变更

启用活动跟踪时，插件会在您的用户模式中添加一个字段：

```ts
user: {
  fields: {
    lastActiveAt: {
      type: "date",
    },
  },
}
```

启用活动跟踪后，请务必运行数据库迁移。

## 客户端设置

```ts
import { createAuthClient } from "better-auth/client";
import { dashClient } from "@better-auth/infra/client";

export const authClient = createAuthClient({
  plugins: [dashClient()],
});
```

### 客户端配置

```ts
dashClient({
  resolveUserId: ({ userId, user, session }) => {
    return userId || user?.id || session?.user?.id;
  },
}),
```

## Dashboard 插件的功能

启用 `dash()` 后，Better Auth 基础设施仪表盘为您提供：

* **用户管理** — 查看、搜索、封禁和删除用户
* **会话监控** — 查看活跃会话并撤销它们
* **组织概览** — 管理组织和成员
* **分析功能** — 跟踪注册、登录和活跃用户随时间的变化
* **审计日志** — 查询事件历史记录（[了解更多](/docs/infrastructure/plugins/audit-logs)）

## 最佳实践

1. **始终设置 API 密钥** — 没有它，插件无法与基础设施 API 通信。

2. **合理使用活动跟踪** — 更新间隔会影响数据库写入频率。对于高流量应用，考虑使用更长的间隔。

3. **监控审计日志保留期限** — 不同套餐的保留周期不同，请检查您的套餐限制。

4. **保护您的端点** — 仪表盘端点需要认证。确保您的仪表盘用户拥有适当权限。
