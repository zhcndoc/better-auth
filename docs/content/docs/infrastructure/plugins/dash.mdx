## 安装

```ts
import { betterAuth } from "better-auth";
import { dash } from "@better-auth/infra";

export const auth = betterAuth({
  plugins: [
    dash({
      apiUrl: process.env.BETTER_AUTH_API_URL,
      kvUrl: process.env.BETTER_AUTH_KV_URL,
      apiKey: process.env.BETTER_AUTH_API_KEY,
    }),
  ],
});
```

## 配置选项

### DashOptions

| 选项 | 类型 | 说明 |
|--------|------|-------------|
| `apiUrl` | `string` | Better Auth 基础设施 API 地址 |
| `kvUrl` | `string` | 用于缓存的 KV 存储地址 |
| `apiKey` | `string` | 你的 API 密钥，用于身份认证 |
| `activityTracking` | `object` | 活动跟踪配置 |

### 活动跟踪

```ts
dash({
  apiKey: process.env.BETTER_AUTH_API_KEY,
  activityTracking: {
    enabled: true,           // 启用活动跟踪
    updateInterval: 300000,  // 更新间隔（毫秒），默认：5 分钟
  },
}),
```

启用后，会在用户 schema 中自动添加并更新 `lastActiveAt` 字段，记录用户最后活动时间。

## 客户端集成

### dashClient()

客户端插件支持审计日志查询功能：

```ts
import { createAuthClient } from "better-auth/client";
import { dashClient } from "@better-auth/infra/client";

export const authClient = createAuthClient({
  plugins: [dashClient()],
});
```

### 配置

```ts
dashClient({
  resolveUserId: ({ userId, user, session }) => {
    // 自定义用户 ID 解析逻辑
    return userId || user?.id || session?.user?.id;
  },
}),
```

### 拉取审计日志

```ts
const session = await authClient.getSession();

const logs = await authClient.dash.getAuditLogs({
  session: session.data,
  limit: 50,
  offset: 0,
});

// 访问数据
console.log(logs.data?.events);  // 审计日志事件数组
console.log(logs.data?.total);   // 总数
console.log(logs.data?.limit);   // 页面大小
console.log(logs.data?.offset);  // 当前偏移量
```

### 查询参数

| 参数 | 类型 | 说明 |
|-----------|------|-------------|
| `limit` | `number` | 每页结果数 (最大 100，默认 50) |
| `offset` | `number` | 分页偏移 (默认 0) |
| `organizationId` | `string` | 按组织过滤 |
| `identifier` | `string` | 按标识符过滤 |
| `eventType` | `string` | 按事件类型过滤 |
| `userId` | `string` | 按用户 ID 过滤 |
| `user` | `object` | 带 ID 的用户对象 |
| `session` | `object` | 带用户信息的会话对象 |

### 分页示例

```ts
async function getAllAuditLogs(session: unknown) {
  const limit = 100;
  let offset = 0;
  const allEvents = [];

  while (true) {
    const result = await authClient.dash.getAuditLogs({
      session,
      limit,
      offset,
    });
    
    const events = result.data?.events ?? [];
    allEvents.push(...events);

    if (events.length < limit) break;
    offset += limit;
  }

  return allEvents;
}
```

### 过滤示例

```ts
// 按事件类型过滤
const signIns = await authClient.dash.getAuditLogs({
  session,
  eventType: "user_signed_in",
});

// 按组织过滤
const orgLogs = await authClient.dash.getAuditLogs({
  session,
  organizationId: "org_123",
});

// 组合过滤
const orgSignIns = await authClient.dash.getAuditLogs({
  session,
  organizationId: "org_123",
  eventType: "user_signed_in",
});
```

## Schema 扩展

启用活动跟踪后，插件会在用户 schema 中添加字段：

```ts
user: {
  fields: {
    lastActiveAt: {
      type: "date",
    },
  },
}
```

启用活动跟踪后，请确保执行数据库迁移。

## 最佳实践

1. **务必设置 API 密钥** —— 没有它，插件无法与基础设施 API 通信。

2. **合理使用活动跟踪** —— 更新间隔影响数据库写入。在高流量应用中，建议使用较长间隔。

3. **关注审计日志保留时间** —— 不同套餐有不同的保留期限，请确认你的套餐。

4. **保护你的端点安全** —— 仪表盘端点需身份验证，确保仪表盘用户拥有相应权限。
