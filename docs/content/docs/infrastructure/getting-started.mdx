<Steps>

  <Step>
    ## 前提条件

    开始之前，请确保您已经拥有：

    1. 一个可用的 [Better Auth](/docs/installation) 安装实例
    2. 来自 [Better Auth 基础设施](/dashboard) 仪表盘的账号和 API 密钥
  </Step>

  <Step>
    ## 安装

    安装 `@better-auth/infra` 包：

    ```package-install
    @better-auth/infra
    ```
  </Step>

  <Step>
    ## 环境变量

    一旦您从基础设施仪表盘获取到 API 密钥，
    请在生产环境变量中添加 `BETTER_AUTH_API_KEY` 环境变量。

    ```dotenv
    # 必填：您的 Better Auth 基础设施 API 密钥
    BETTER_AUTH_API_KEY=your_api_key_here
    ```

    <Callout>
      您可以通过注册并在
      [Better Auth 基础设施仪表盘](/dashboard) 创建新项目来获取您的 API 密钥。
    </Callout>
  </Step>

</Steps>


## 服务器设置

### 基础配置

将 `dash()` 插件添加到您的 Better Auth 配置中：

```ts
import { betterAuth } from "better-auth";
import { dash } from "@better-auth/infra"; // [!code highlight]

export const auth = betterAuth({
  // ... 您现有的 Better Auth 配置
  plugins: [
    dash({ // [!code highlight]
      apiKey: process.env.BETTER_AUTH_API_KEY, // [!code highlight]
    }), // [!code highlight]
  ],
});
```

如果您订阅的是 **pro** 计划及以上，您可以使用 `sentinel()` 插件启用安全检查、滥用防护等功能：

```ts
import { betterAuth } from "better-auth";
import { dash, sentinel } from "@better-auth/infra";  // [!code highlight]

export const auth = betterAuth({
  plugins: [
    dash({ /* ... */ }),
    sentinel({ // [!code highlight]
      apiKey: process.env.BETTER_AUTH_API_KEY, // [!code highlight]
    }) // [!code highlight]
  ],
});
```

## 客户端设置

### 基础客户端配置

将客户端插件添加到您的认证客户端中：

```ts
import { createAuthClient } from "better-auth/client";
import { dashClient, sentinelClient } from "@better-auth/infra/client";

export const authClient = createAuthClient({
  plugins: [
    dashClient(),
    sentinelClient({
      autoSolveChallenge: true, // 自动解决 PoW 挑战
    }),
  ],
});
```



### 插件概览

- **`dash()`** - 启用分析追踪、审计日志、仪表盘管理 API 等功能
- **`sentinel()`** - 启用安全检查和滥用防护

您可以单独使用其中任何一个插件，但同时使用两者可获得完整的 Better Auth 基础设施体验。
