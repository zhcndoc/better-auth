<Steps>
    <Step>
        ### 获取你的 Vercel 凭证
        要使用 Vercel 登录（Sign in with Vercel），你需要一个客户端 ID 和客户端密钥。你可以通过 [Vercel 控制台](https://vercel.com/dashboard) 通过 [创建 Vercel 应用](https://vercel.com/docs/sign-in-with-vercel/manage-from-dashboard) 获取它们。

        请确保将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/vercel` 用于本地开发。生产环境则应设置为你应用的 URL。如果你更改了认证路由的基础路径，也应相应更新重定向 URL。

        <Callout type="info">
            Vercel 需要 PKCE（Proof Key for Code Exchange，代码交换证明密钥）以增强安全性。Better Auth 会自动处理这一点。
        </Callout>
    </Step>

  <Step>
        ### 配置提供者
        要配置提供者，你需要导入该提供者并将其传递给认证实例的 `socialProviders` 选项。

        ```ts title="auth.ts" 
        import { betterAuth } from "better-auth"

        export const auth = betterAuth({ 
            socialProviders: {
                vercel: { // [!code highlight]
                    clientId: process.env.VERCEL_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.VERCEL_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            },
        })
        ```
    </Step>
</Steps>

## 使用方法

### 使用 Vercel 登录

要使用 Vercel 登录，你可以使用客户端提供的 `signIn.social` 函数。`signIn` 函数接收一个对象，包含以下属性：

- `provider`：要使用的提供者，应设置为 `vercel`。

```ts title="auth-client.ts"
import { createAuthClient } from "better-auth/client"
const authClient = createAuthClient()

const signIn = async () => {
    const data = await authClient.signIn.social({
        provider: "vercel"
    })
}
```

## 选项

有关所有社交提供者支持的完整选项列表，请查看 [提供者选项](/docs/concepts/oauth#provider-options)。

### 可用作用域

Vercel 支持以下 OpenID Connect 作用域：

- `openid`（默认）：在 ID 令牌中返回用户的唯一标识符
- `email`：返回用户的电子邮件地址
- `profile`：返回用户的个人资料信息（姓名、头像）
- `offline_access`：返回用于离线访问的刷新令牌

作用域在 Vercel 应用中配置，因此无需向提供者传递 `scope` 参数，且省略此参数更为方便。

如果你传递了 `scope` 参数，则表示请求已配置作用域的一个子集：

```ts title="auth.ts"
import { betterAuth } from "better-auth"

export const auth = betterAuth({
    socialProviders: {
        vercel: {
            clientId: process.env.VERCEL_CLIENT_ID as string,
            clientSecret: process.env.VERCEL_CLIENT_SECRET as string,
            scope: ["openid", "email", "profile"], // [!code highlight]
        },
    },
})
```

    更多关于 Vercel 作用域及 API 功能的信息，请参阅 [官方 Vercel 文档](https://vercel.com/docs/sign-in-with-vercel)。
</Callout>
