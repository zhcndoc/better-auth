<Steps>
    <Step>
        ### 获取你的 PayPal 凭证
        要集成 PayPal，您需要通过在 [PayPal 开发者门户](https://developer.paypal.com/dashboard) 创建应用程序来获取 API 凭证。

         请按以下步骤操作：
            1. 在 PayPal 开发者门户创建一个账号
            2. 创建一个新的应用程序，[官方文档](https://developer.paypal.com/developer/applications/)
            3. 在“其他功能”中配置“使用 PayPal 登录”
            4. 设置您的返回 URL（重定向 URL）
            5. 配置用户信息权限
            6. 记录您的客户端 ID 和客户端密钥

        <Callout type="info">
            - PayPal 有两个环境：沙箱（用于测试）和正式环境（生产）
            - 测试时，请在开发者仪表盘的“沙箱” → “账户”中创建沙箱测试账户
            - 不能使用真实 PayPal 账号在沙箱模式进行测试，必须使用生成的测试账户
            - PayPal 应用设置里的返回 URL 必须与您的重定向 URI 完全匹配
            - PayPal API 不支持 localhost。您需要为重定向 URL 使用公共域名，并且本地测试需使用 HTTPS。您可以使用 [NGROK](https://ngrok.com/) 或其他类似工具实现此目的。
        </Callout>
         请确保在应用设置中配置“使用 PayPal 登录”：
            1. 在开发者仪表盘进入您的应用
            2. 在“其他功能”中勾选“使用 PayPal 登录”
            3. 点击“高级设置”
            4. 输入您的返回 URL
            5. 选择您想访问的用户信息（邮箱、姓名等）
            6. 输入隐私政策和用户协议的 URL

        <Callout type="info">
            - PayPal 在授权 URL 中不使用传统的 OAuth2 作用域，而是在开发者仪表盘直接配置权限
            - 对于正式应用，PayPal 必须审核并批准您的应用，通常需要几周时间
        </Callout>
    </Step>

  <Step>
        ### 配置提供商
        要配置提供商，您需要导入该提供商，并将其传递给 auth 实例的 `socialProviders` 选项。

        ```ts title="auth.ts"
        import { betterAuth } from "better-auth"

        export const auth = betterAuth({
            socialProviders: {
                paypal: { // [!code highlight]
                    clientId: process.env.PAYPAL_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.PAYPAL_CLIENT_SECRET as string, // [!code highlight]
                    environment: "sandbox", // 或者生产环境为 "live" // [!code highlight]
                }, // [!code highlight]
            },
        })
        ```
        #### 选项
        PayPal 提供商接受以下选项：
    
        - `environment`: `'sandbox' | 'live'` - 使用的 PayPal 环境（默认: `'sandbox'`）
        - `requestShippingAddress`: `boolean` - 是否请求配送地址信息（默认: `false`）
    
        ```ts title="auth.ts"
        export const auth = betterAuth({
            socialProviders: {
                paypal: {
                    clientId: process.env.PAYPAL_CLIENT_ID as string,
                    clientSecret: process.env.PAYPAL_CLIENT_SECRET as string,
                    environment: "live", // 正式环境使用 "live"
                    requestShippingAddress: true, // 请求地址信息
                },
            },
        })
        ```
    </Step>
       <Step>
        ### 使用 PayPal 登录
        登录时，您可以使用客户端提供的 `signIn.social` 函数。`signIn` 函数接受一个包含以下属性的对象：
        - `provider`：要使用的提供商，应设置为 `paypal`。

        ```ts title="auth-client.ts"
        import { createAuthClient } from "better-auth/client"
        const authClient =  createAuthClient()

        const signIn = async () => {
            const data = await authClient.signIn.social({
                provider: "paypal"
            })
        }
        ```
        ### 其他选项：
        - `environment`：使用的 PayPal 环境。
            - 默认值: `"sandbox"`
            - 可选项: `"sandbox"` | `"live"`
        - `requestShippingAddress`：是否请求配送地址信息。
            - 默认值: `false`
        - `scope`：要请求的额外权限（与默认权限合并）。
            - 默认值：在 PayPal 开发者仪表盘中配置
            - 注意：PayPal 不使用传统的 OAuth2 作用域 — 权限在仪表盘设置
            详情请参阅 [权限参考](https://developer.paypal.com/docs/log-in-with-paypal/integrate/reference/#scope-attributes)
        - `mapProfileToUser`：自定义函数，用于将 PayPal 资料数据映射到用户对象。
        - `getUserInfo`：自定义函数，用于获取用户信息。
        详情请参阅 [用户参考](https://developer.paypal.com/docs/api/identity/v1/#userinfo_get)
        - `verifyIdToken`：自定义 ID 令牌验证函数。   
    </Step>

</Steps>
