---
title: Cognito
description: Amazon Cognito 提供者的设置与使用。
---

<Steps>
  <Step>
    ### 获取你的 Cognito 凭证
    要集成 Cognito，你需要在 [Amazon Cognito 控制台](https://console.aws.amazon.com/cognito/) 中设置一个 **用户池（User Pool）** 和一个 **应用客户端（App client）**。

    按照以下步骤操作：
    1. 打开 **Cognito 控制台** 并创建一个 **用户池**。
    2. 在 **应用客户端** 中，创建一个新的 **应用客户端** （如果启用，请记录客户端 ID 和客户端密钥）。
    3. 进入 **域名** 页面，设置一个 Cognito 托管 UI 域名（例如：`your-app.auth.us-east-1.amazoncognito.com`）。
    4. 在 **应用客户端设置** 中，启用：
       - 允许的 OAuth 流程：`Authorization code grant`
       - 允许的 OAuth 作用域：`openid`、`profile`、`email`
    5. 添加你的回调 URL（例如，`http://localhost:3000/api/auth/callback/cognito`）。

    <Callout type="info">
      - **用户池是 Cognito 认证的必备条件**。  
      - 确保回调 URL 与你在 Cognito 中配置的完全一致。  
    </Callout>
  </Step>

  <Step>
    ### 配置提供者
    在你的 `auth` 实例中，配置 `socialProviders` 的 `cognito` 键。

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth";

    export const auth = betterAuth({
      socialProviders: {
        cognito: {
          clientId: process.env.COGNITO_CLIENT_ID as string, // [!code highlight]
          clientSecret: process.env.COGNITO_CLIENT_SECRET as string, // [!code highlight]
          domain: process.env.COGNITO_DOMAIN as string, // 例如 "your-app.auth.us-east-1.amazoncognito.com" [!code highlight]
          region: process.env.COGNITO_REGION as string, // 例如 "us-east-1" [!code highlight]
          userPoolId: process.env.COGNITO_USERPOOL_ID as string, // [!code highlight]
        },
      },
    })
    ```
  </Step>

  <Step>
    ### 使用 Cognito 登录
    要使用 Cognito 登录，请从客户端调用 `signIn.social` 函数。  

    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client"

    const authClient = createAuthClient()

    const signIn = async () => {
      const data = await authClient.signIn.social({
        provider: "cognito"
      })
    }
    ```
    
      ### 额外选项：
        - `scope`：需要请求的额外 OAuth2 作用域（与默认权限合并）。
            - 默认值：`"openid" "profile" "email"`
            - 常见的 Cognito 作用域：
              - `openid`：OpenID Connect 认证必需
              - `profile`：访问基本的个人资料信息
              - `email`：访问用户的邮箱
              - `phone`：访问用户的电话号码
              - `aws.cognito.signin.user.admin`：允许访问 Cognito 特定的 API
        - 注意：你必须在 Cognito 应用客户端设置中配置这些作用域。[可用作用域](https://docs.aws.amazon.com/cognito/latest/developerguide/token-endpoint.html#token-endpoint-userinfo)
        - `getUserInfo`：自定义函数，用于从 Cognito 的 UserInfo 端点获取用户信息。  
       <Callout type="info">
        有关 Amazon Cognito 作用域和 API 功能的更多信息，请参阅 [官方文档](https://docs.aws.amazon.com/cognito/latest/developerguide/cognito-user-pools-define-resource-servers.html?utm_source)。
        </Callout>
  </Step>
</Steps>
