<Steps>
    <Step> 
        ### 获取你的 Google 凭据
        要使用 Google 作为社交提供者，你需要获取你的 Google 凭据。你可以通过在 [Google 云控制台](https://console.cloud.google.com/apis/dashboard) 创建一个新项目来获得。

        在 Google 云控制台 > 凭据 > 授权重定向 URI，确保在本地开发时将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/google`。在生产环境中，请确保将重定向 URL 设置为你的应用域名，例如 `https://example.com/api/auth/callback/google`。如果你更改了认证路由的基础路径，应相应更新重定向 URL。

        <Callout type="info">
        **创建你的 Google OAuth 凭据**

        如果你还没有创建 OAuth 凭据，请按照以下步骤操作：

        1. 打开 **Google 云控制台** → **API 与服务** → **凭据**
        2. 点击 **创建凭据** → **OAuth 客户端 ID**
        3. 选择 **Web 应用**
        4. 添加你的重定向 URI:
           - `http://localhost:3000/api/auth/callback/google`（用于本地开发）
           - `https://your-domain.com/api/auth/callback/google`（用于生产）
        5. 将 **客户端 ID** 和 **客户端密钥** 复制到你的环境变量中

        这些步骤可以避免常见的 `redirect_uri_mismatch` 问题。
        </Callout>
    </Step>

  <Step>
        ### 配置提供者
        要配置提供者，你需要在认证配置中向 `socialProviders.google` 传递 `clientId` 和 `clientSecret`。

        ```ts title="auth.ts"
        import { betterAuth } from "better-auth"

        export const auth = betterAuth({
            baseURL: process.env.BETTER_AUTH_URL, // [!code highlight]
            socialProviders: {
                google: { // [!code highlight]
                    clientId: process.env.GOOGLE_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.GOOGLE_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            },
        })
        ```
        <Callout type="warn">
        **重要：设置你的基础 URL**

        你必须配置 `baseURL` 来避免 `redirect_uri_mismatch` 错误。Better Auth 使用它来构建发送给 Google 的 OAuth 回调 URL。

        **方案一：环境变量（推荐）**

        在你的 `.env` 文件中添加：
        ```dotenv
        BETTER_AUTH_URL=https://your-domain.com
        ```

        **方案二：显式配置**

        如上所示，直接在认证配置中传递 `baseURL`。
        如果不这样做，回调 URL 可能默认为 `localhost`，导致 Google OAuth 在生产环境中失败。
        </Callout>
    </Step>

</Steps>

## 使用方法

### 使用 Google 登录

要使用 Google 登录，可以使用客户端提供的 `signIn.social` 函数。`signIn` 函数接收一个带有以下属性的对象：

- `provider`：要使用的提供者，应设置为 `google`。

```ts title="auth-client.ts"  /
import { createAuthClient } from "better-auth/client";
const authClient = createAuthClient();

const signIn = async () => {
  const data = await authClient.signIn.social({
    provider: "google",
  });
};
```

### 使用 ID Token 登录 Google

要使用 Google 的 ID Token 登录，可以通过 `signIn.social` 函数传递 ID Token。

当你在客户端已经有 Google 的 ID Token 并想用它在服务器端登录时，这很有用。

<Callout>
  如果提供了 ID Token，将不会发生重定向，用户会直接登录。
</Callout>

```ts title="auth-client.ts"
const data = await authClient.signIn.social({
    provider: "google",
    idToken: {
        token: // Google ID Token,
        accessToken: // Google 访问令牌
    }
})
```

<Callout>
  如果你想使用 Google 一键登录，可以参考 [One Tap 插件](/docs/plugins/one-tap) 指南。
</Callout>

### 始终要求选择账户

如果你想始终要求用户选择账户，可以向提供者传递 `prompt` 参数，设置为 `select_account`。

```ts
socialProviders: {
    google: {
        prompt: "select_account", // [!code highlight]
        clientId: process.env.GOOGLE_CLIENT_ID as string,
        clientSecret: process.env.GOOGLE_CLIENT_SECRET as string,
    },
}
```

### 请求额外的 Google 权限

如果你的应用需要用户已经注册后访问额外的 Google 权限（例如 Google Drive、Gmail 或其他 Google 服务），你可以使用相同的 Google 提供者通过 `linkSocial` 方法请求这些权限。

```tsx title="auth-client.ts"
const requestGoogleDriveAccess = async () => {
  await authClient.linkSocial({
    provider: "google",
    scopes: ["https://www.googleapis.com/auth/drive.file"],
  });
};

// 在 React 组件中的示例用法
return (
  <button onClick={requestGoogleDriveAccess}>
    添加 Google Drive 权限
  </button>
);
```

这将触发新的 OAuth 流程，请求额外权限。完成后，你的账户将在数据库中拥有新的权限，访问令牌将允许你访问请求的 Google API。

<Callout>
  确保你使用的是 Better Auth 版本 1.2.7 或更高版本，以避免请求同一提供者额外权限时出现 “社交账户已绑定” 错误。
</Callout>

### 始终获取刷新令牌

Google 只在用户第一次同意授权你的应用时发放刷新令牌。
如果用户已经授权过，后续 OAuth 流程只会返回访问令牌，不会返回刷新令牌。

要始终获取刷新令牌，你可以在提供者选项中设置 `accessType` 为 `offline` ，并将 `prompt` 设置为 `select_account consent`。

```ts
socialProviders: {
    google: {
        clientId: process.env.GOOGLE_CLIENT_ID as string,
        clientSecret: process.env.GOOGLE_CLIENT_SECRET as string,
        accessType: "offline", // [!code highlight]
        prompt: "select_account consent", // [!code highlight]
    },
}
```

<Callout>
  **撤销访问权限：** 如果你想为已经授权你的应用的用户获取新的刷新令牌，用户必须先在其 Google 账户设置中撤销你的应用访问权限，然后重新授权。
</Callout>
