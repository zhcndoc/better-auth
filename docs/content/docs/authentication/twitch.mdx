---
title: Twitch
description: Twitch 提供者设置和使用。
---

<Steps>
    <Step> 
        ### 获取您的 Twitch 凭据
        要使用 Twitch 登录，您需要客户端 ID 和客户端密钥。您可以从 [Twitch 开发者门户](https://dev.twitch.tv/console/apps) 获取它们。

        确保将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/twitch` 以进行本地开发。对于生产环境，您应将其设置为您的应用程序的 URL。如果您更改了身份验证路由的基础路径，也应相应更新重定向 URL。
    </Step>

  <Step>
        ### 配置提供者
        要配置提供者，您需要导入该提供者并将其传递给认证实例的 `socialProviders` 选项。

        ```ts title="auth.ts"  
        import { betterAuth } from "better-auth"
        
        export const auth = betterAuth({
            socialProviders: {
                twitch: { // [!code highlight]
                    clientId: process.env.TWITCH_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.TWITCH_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            }
        })
        ```
    </Step>
       <Step>
        ### 使用 Twitch 登录
        要使用 Twitch 登录，您可以使用客户端提供的 `signIn.social` 函数。
        `signIn` 函数接收一个包含以下属性的对象：

        - `provider`: 要使用的提供者。应设置为 `twitch`。

        ```ts title="auth-client.ts"  
        import { createAuthClient } from "better-auth/client"
        const authClient =  createAuthClient()
        
        const signIn = async () => {
            const data = await authClient.signIn.social({
                provider: "twitch"
            })
        }
        ```

        <Callout type="warn">
            没有电子邮件地址的 Twitch 用户将无法登录。
        </Callout>
    </Step>
</Steps>
