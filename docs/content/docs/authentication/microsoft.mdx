启用 Microsoft Azure Entra ID（曾称 Active Directory）OAuth，允许您的用户使用其 Microsoft 账号登录和注册您的应用。

<Steps>
  <Step>
    ### 获取 Microsoft 凭据

    要将 Microsoft 用作社交提供商，您需要获取 Microsoft 凭据。这涉及使用您的 Microsoft Entra ID 仪表板账户生成自己的客户端 ID 和客户端密钥。

    请确保将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/microsoft` 用于本地开发。对于生产环境，请将其更改为您的应用程序 URL。如果更改了认证路由的基础路径，请相应更新重定向 URL。

    有关更多信息，请参阅 [Microsoft Entra ID 文档](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-register-app)。
  </Step>

  <Step>
    ### 配置提供商

    要配置提供商，您需要在认证配置中将 `clientId` 和 `clientSecret` 传递给 `socialProviders.microsoft`。

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth"

    export const auth = betterAuth({
        socialProviders: {
            microsoft: { // [!code highlight]
                clientId: process.env.MICROSOFT_CLIENT_ID as string, // [!code highlight]
                clientSecret: process.env.MICROSOFT_CLIENT_SECRET as string, // [!code highlight]
                // 可选
                tenantId: 'common', // [!code highlight]                
                authority: "https://login.microsoftonline.com", // 认证授权 URL // [!code highlight]
                prompt: "select_account", // 强制选择账号 // [!code highlight]
            }, // [!code highlight]
        },
    })
    ```

    **Authority URL**: 标准 Entra ID 场景使用默认的 `https://login.microsoftonline.com`，CIAM（客户标识与访问管理）场景使用 `https://<tenant-id>.ciamlogin.com`。

    <Callout type="warn">
      默认情况下，Entra 不会为受管用户发出 `email` 声明，并且该值[可由租户变更且从不由 Microsoft 验证](https://learn.microsoft.com/en-us/entra/identity-platform/id-token-claims-reference)；它绝不能用于授权决策。对于受管用户，请将 `email` 作为[可选声明](https://learn.microsoft.com/en-us/entra/identity-platform/optional-claims)请求，并使用 `profile.oid`（如果需要跨租户关联，则再加上 `profile.tid`）作为稳定的身份锚点。有关 `mapProfileToUser` 的回退方式，请参阅[处理不提供电子邮件的提供商](/docs/concepts/oauth#handling-providers-without-email)。
    </Callout>
  </Step>
</Steps>

## 使用 Microsoft 登录

要使用 Microsoft 登录，您可以调用客户端提供的 `signIn.social` 函数。`signIn` 函数接收一个包含以下属性的对象：

* `provider`：要使用的提供商，应设置为 `microsoft`。

```ts title="auth-client.ts"
import { createAuthClient } from "better-auth/client";

const authClient = createAuthClient();

const signIn = async () => {
  const data = await authClient.signIn.social({
    provider: "microsoft",
    callbackURL: "/dashboard", // 登录后跳转的 URL
  });
};
```
