title: 其他社交登录提供商
description: 其他社交登录提供商的设置和使用。
---

Better Auth 支持任何实现了 OAuth2 协议或 OpenID Connect（OIDC）流程的社交登录提供商，通过 [Generic OAuth 插件](/docs/plugins/generic-oauth) 实现。你可以使用为常见提供商（如 Auth0、Keycloak、Okta、Microsoft Entra ID 和 Slack）预配置的辅助函数，或者手动配置任何 OAuth 提供商。

## 安装

<Steps>
  <Step>
    ### 将插件添加到你的认证配置中

    要使用 Generic OAuth 插件，需要将其添加到你的认证配置中。

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth"
    import { genericOAuth } from "better-auth/plugins" // [!code highlight]

    export const auth = betterAuth({
        // ... 其他配置选项
        plugins: [
            genericOAuth({ // [!code highlight]
                config: [ // [!code highlight]
                    { // [!code highlight]
                        providerId: "provider-id", // [!code highlight]
                        clientId: "test-client-id", // [!code highlight]
                        clientSecret: "test-client-secret", // [!code highlight]
                        discoveryUrl: "https://auth.example.com/.well-known/openid-configuration", // [!code highlight]
                        // ... 其他配置选项 // [!code highlight]
                    }, // [!code highlight]
                    // 根据需要添加更多提供商 // [!code highlight]
                ] // [!code highlight]
            }) // [!code highlight]
        ]
    })
    ```

  </Step>

  <Step>
    ### 添加客户端插件

    在你的认证客户端实例中包含 Generic OAuth 客户端插件。

    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client"
    import { genericOAuthClient } from "better-auth/client/plugins"

    const authClient = createAuthClient({
        plugins: [
            genericOAuthClient()
        ]
    })
    ```

  </Step>
</Steps>

<Callout>
  详细了解 Generic OAuth 插件的安装和使用请见
  [这里](/docs/plugins/generic-oauth#usage)。
</Callout>

## 使用示例

以下是配置通用 OAuth 提供商的基本示例：

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { genericOAuth } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    genericOAuth({
      config: [
        {
          providerId: "provider-id",
          clientId: process.env.CLIENT_ID,
          clientSecret: process.env.CLIENT_SECRET,
          discoveryUrl: "https://auth.example.com/.well-known/openid-configuration",
        },
      ],
    }),
  ],
})
```

## 使用预配置提供商

Better Auth 为流行的 OAuth 提供商提供了预配置的辅助函数。下面是使用 Slack 的示例：

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { genericOAuth, slack } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    genericOAuth({
      config: [
        slack({
          clientId: process.env.SLACK_CLIENT_ID,
          clientSecret: process.env.SLACK_CLIENT_SECRET,
        }),
      ],
    }),
  ],
})
```

```ts title="sign-in.ts"
const response = await authClient.signIn.oauth2({
  providerId: "slack",
  callbackURL: "/dashboard",
})
```

有关更多预配置提供商（Auth0、Keycloak、Okta、Microsoft Entra ID）及其配置选项，请参阅 [Generic OAuth 插件文档](/docs/plugins/generic-oauth#pre-configured-provider-helpers)。

## 手动配置示例

如果你需要配置没有预定义辅助函数的提供商，可以手动进行配置：

### Instagram 示例

```ts title="auth.ts"
import { betterAuth } from "better-auth";
import { genericOAuth } from "better-auth/plugins";

export const auth = betterAuth({
  // ... 其他配置选项
  plugins: [
    genericOAuth({
      config: [
        {
          providerId: "instagram",
          clientId: process.env.INSTAGRAM_CLIENT_ID as string,
          clientSecret: process.env.INSTAGRAM_CLIENT_SECRET as string,
          authorizationUrl: "https://api.instagram.com/oauth/authorize",
          tokenUrl: "https://api.instagram.com/oauth/access_token",
          scopes: ["user_profile", "user_media"],
        },
      ],
    }),
  ],
});
```

```ts title="sign-in.ts"
const response = await authClient.signIn.oauth2({
  providerId: "instagram",
  callbackURL: "/dashboard", // 用户认证成功后的重定向路径
});
```

```ts title="auth.ts"
import { betterAuth } from "better-auth";
import { genericOAuth } from "better-auth/plugins";

export const auth = betterAuth({
  // ... 其他配置选项

      ],
    }),
  ],
});
```

```ts title="sign-in.ts"
const response = await authClient.signIn.oauth2({

```
