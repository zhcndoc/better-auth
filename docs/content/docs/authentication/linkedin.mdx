<Steps>
    <Step> 
        ### 获取你的 LinkedIn 凭据
        要使用 LinkedIn 登录，你需要客户端 ID 和客户端密钥。你可以从 [LinkedIn 开发者门户](https://www.linkedin.com/developers/) 获取它们。
        
        确保将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/linkedin` 用于本地开发。对于生产环境，你应该设置为你的应用程序的 URL。如果你更改了认证路由的基础路径，也应相应更新重定向 URL。
    </Step>

    <Callout type="info">
     在 LinkedIn 门户的产品中，你需要**使用 OpenID Connect 的 LinkedIn 登录**产品。
    </Callout>

    这里有一些不同的指南：
        [授权码流程（3步 OAuth）（过时）](https://learn.microsoft.com/en-us/linkedin/shared/authentication/authorization-code-flow)
        [使用 OpenID Connect 的 LinkedIn 登录](https://learn.microsoft.com/en-us/linkedin/consumer/integrations/self-serve/sign-in-with-linkedin-v2?context=linkedin%2Fconsumer%2Fcontext)

  <Step>
        ### 配置提供商
        要配置提供商，你需要导入提供商并将其传递给认证实例的 `socialProviders` 选项。

        ```ts title="auth.ts"
        import { betterAuth } from "better-auth"

        export const auth = betterAuth({
            socialProviders: {
                linkedin: { // [!code highlight]
                    clientId: process.env.LINKEDIN_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.LINKEDIN_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            },
        })
        ```
    </Step>
       <Step>
        ### 使用 LinkedIn 登录
        要使用 LinkedIn 登录，你可以使用客户端提供的 `signIn.social` 函数。`signIn` 函数接受一个包含以下属性的对象：
        - `provider`: 要使用的提供商，应设置为 `linkedin`。

        ```ts title="auth-client.ts"
        import { createAuthClient } from "better-auth/client"
        const authClient =  createAuthClient()

        const signIn = async () => {
            const data = await authClient.signIn.social({
                provider: "linkedin"
            })
        }
        ```
    </Step>

</Steps>
