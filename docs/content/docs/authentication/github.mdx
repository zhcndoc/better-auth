<Steps>
    <Step> 
        ### 获取你的 GitHub 凭据
        要使用 GitHub 登录，你需要一个客户端 ID 和客户端密钥。你可以从 [GitHub 开发者门户](https://github.com/settings/developers) 获取它们。

        确保将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/github` 以便本地开发。对于生产环境，你应该将其设置为你的应用程序的 URL。如果你更改了认证路由的基础路径，也应相应更新重定向 URL。

        重要：你必须在你的 GitHub 应用中包含 user:email 权限范围。详情见下。
    </Step>

  <Step>
        ### 配置提供者
        要配置提供者，你需要导入该提供者并将其传递给认证实例的 `socialProviders` 选项。

        ```ts title="auth.ts"
        import { betterAuth } from "better-auth"
        
        export const auth = betterAuth({
            socialProviders: {
                github: { // [!code highlight]
                    clientId: process.env.GITHUB_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.GITHUB_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            },
        })
        ```
    </Step>
       <Step>
        ### 使用 GitHub 登录
        要使用 GitHub 登录，可以调用客户端提供的 `signIn.social` 函数。`signIn` 函数接收一个带有以下属性的对象：
        - `provider`：要使用的提供者，应设置为 `github`。

        ```ts title="auth-client.ts"  
        import { createAuthClient } from "better-auth/client"
        const authClient =  createAuthClient()
        
        const signIn = async () => {
            const data = await authClient.signIn.social({
                provider: "github"
            })
        }
        ```
    </Step>
</Steps>

## 使用说明

### 设置你的 Github 应用

Github 有两种应用类型：Github 应用和 OAuth 应用。

对于 OAuth 应用，你无需做特殊操作（只需按照上述步骤操作）。对于 Github 应用，你需要额外完成一项设置，即开启读取用户邮箱的权限：

1. 创建应用后，进入 *Permissions and Events* > *Account Permissions* > *Email Addresses* 并选择“只读”（Read-Only）

2. 保存更改。

就是这样！现在你可以复制你的应用的客户端 ID 和客户端密钥了！

<Callout>
如果你遇到 "email_not_found" 错误，说明你选择的是 Github 应用但没有配置此部分！
</Callout>

### 为什么我没有刷新令牌？

Github 不为 OAuth 应用发放刷新令牌。对于常规 OAuth 应用，GitHub 发放的访问令牌除非用户撤销、应用撤销，或者一年内未使用，否则将一直有效。

不需要刷新令牌是因为访问令牌不像 Google 或 Discord 那样在短时间内过期。
