---
title: Kakao
description: Kakao 提供者的设置与使用。
---

<Steps>
    <Step> 
        ### 获取您的 Kakao 凭据
        要使用 Kakao 登录，您需要客户端 ID 和客户端密钥。您可以从 [Kakao 开发者门户](https://developers.kakao.com) 获取它们。

        确保将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/kakao` 用于本地开发。生产环境下，您应将其设置为您应用程序的 URL。如果您更改了认证路由的基础路径，您应相应更新重定向 URL。
    </Step>
    <Step>
        ### 配置提供者
        要配置提供者，您需要导入提供者并传递给认证实例的 `socialProviders` 选项。

        <Callout type="info">
            - 默认的权限范围（scopes）为 `account_email`、`profile_image` 和 `profile_nickname`。
            - 请注意，获取 `account_email` 需要应用为 **Biz App**（已完成企业验证的应用）。更多详情请参阅 [Kakao 登录权限文档](https://developers.kakao.com/docs/latest/kakaologin/utilize#scope-user)。
        </Callout>

        ```ts title="auth.ts"
        import { betterAuth } from "better-auth"

        export const auth = betterAuth({
            socialProviders: {
                kakao: { // [!code highlight]
                    clientId: process.env.KAKAO_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.KAKAO_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            }
        })
        ```
    </Step>
    <Step>
        ### 使用 Kakao 登录
        要使用 Kakao 登录，您可以使用客户端提供的 `signIn.social` 函数。`signIn` 函数接受一个包含如下属性的对象：
        - `provider`：要使用的提供者，应设置为 `kakao`。

        ```ts title="auth-client.ts"
        import { createAuthClient } from "better-auth/client"
        const authClient =  createAuthClient()

        const signIn = async () => {
            const data = await authClient.signIn.social({
                provider: "kakao"
            })
        }
        ```
    </Step>
</Steps>
