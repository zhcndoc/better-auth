---
title: Kick
description: Kick 提供者的设置和使用。
---

<Steps>
    <Step> 
        ### 获取你的 Kick 凭据
        要使用 Kick 登录，你需要一个客户端 ID 和客户端密钥。你可以从 [Kick 开发者门户](https://kick.com/settings/developer) 获取它们。

        确保将重定向 URL 设置为 `http://localhost:3000/api/auth/callback/kick` 以便本地开发。对于生产环境，你应该将其设置为你的应用程序的 URL。如果你更改了认证路由的基本路径，应该相应更新重定向 URL。
    </Step>

  <Step>
        ### 配置提供者
        要配置提供者，你需要导入提供者并将其传递给 auth 实例的 `socialProviders` 选项。

        ```ts title="auth.ts"  
        import { betterAuth } from "better-auth"
        
        export const auth = betterAuth({
            socialProviders: {
                kick: { // [!code highlight]
                    clientId: process.env.KICK_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.KICK_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            }
        })
        ```
    </Step>
       <Step>
        ### 使用 Kick 登录
        要使用 Kick 登录，你可以使用客户端提供的 `signIn.social` 函数。`signIn` 函数接收一个包含以下属性的对象：
        - `provider`：使用的提供者，应设置为 `kick`。

        ```ts title="auth-client.ts"  
        import { createAuthClient } from "better-auth/client"
        const authClient =  createAuthClient()
        
        const signIn = async () => {
            const data = await authClient.signIn.social({
                provider: "kick"
            })
        }
        ```
    </Step>
</Steps>
