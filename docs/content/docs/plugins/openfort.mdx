[Openfort](https://www.openfort.io/) 是一个用于将区块链钱包集成到应用中的基础设施提供商。  
它通过嵌入式钱包实现无缝的钱包创建、管理和区块链交互。

此插件由 Openfort 团队维护。有关 Bug、问题或功能请求，请访问 [Openfort Adapters GitHub 仓库](https://github.com/openfort-xyz/openfort-adapters)。

## 功能

- 嵌入式钱包的创建和管理。
- 与 Better Auth 身份验证无缝集成。
- 支持 EVM 链（Ethereum、Polygon、Base 等）和/或 Solana —— 可单独使用或同时使用两者。
- 用户交易的 Gas 资助。
- 多种恢复方式（Passkey、密码、自动恢复）。
- 签名消息及执行智能合约交互。
- 可选的 Wagmi 集成，用于 EVM 链。

## 安装

<Steps>
  <Step>
    ### 安装 Openfort SDK

    #### 服务器端

    ```package-install
    @openfort/openfort-node
    ```

    #### 客户端

    ```package-install
    @openfort/react @tanstack/react-query
    ```

    Then install the peer dependencies for the chains you need:

    **For EVM (Ethereum, Polygon, Base, etc.):**
    ```package-install
    viem wagmi
    ```

    **For Solana:**
    ```package-install
    @solana/kit
    ```

    <Callout>
      `@openfort/react` 包支持 **EVM 链**（Ethereum、Polygon、Base 等）和 **Solana**。你可以构建仅 EVM、仅 Solana 或多链应用。React SDK 通过 Wagmi 实现 EVM 链的区块链交互，Solana 则使用 `@solana/kit`。  
      若需要支持其他链，请直接使用 Openfort JS SDK。
    </Callout>
  </Step>

  <Step>
    ### 安装 Bearer 插件

    Openfort 集成需要 Bearer 插件用于基于令牌的认证。

    ```package-install
    better-auth @openfort/better-auth 
    ```

    Bearer 插件包含在 Better Auth 核心包中。
  </Step>

  <Step>
    ### 配置环境变量

    #### 服务器端 (.env)

    ```bash title=".env"
    # Better Auth
    BETTER_AUTH_SECRET=your-random-secret-here  # 生成方法：openssl rand -base64 32
    BETTER_AUTH_URL=http://localhost:3000

    # Openfort
    OPENFORT_SECRET_KEY=sk_test_...

    # Shield 加密（用于嵌入式钱包）
    SHIELD_PUBLISHABLE_KEY=your-shield-api-key
    SHIELD_ENCRYPTION_SHARE=your-shield-encryption-share
    SHIELD_SECRET_KEY=your-shield-secret-key

    # 前端 URL（用于 CORS）
    FRONTEND_URL=http://localhost:5173
    ```

    #### 客户端 (.env.local)

    ```bash title=".env.local"
    # Openfort
    VITE_OPENFORT_PUBLISHABLE_KEY=pk_test_...
    VITE_SHIELD_PUBLISHABLE_KEY=...
    VITE_FEE_SPONSORSHIP_ID=pol_...  # Openfort 控制面板中的费用资助 ID

    # Better Auth
    VITE_BETTERAUTH_URL=http://localhost:3000
    VITE_BETTERAUTH_BASE_PATH=/api/auth
    ```

    <Callout>
      Openfort API 密钥请从 [Openfort 控制面板](https://dashboard.openfort.io/) 获取。嵌入式钱包加密需要 Shield 凭据。
    </Callout>
  </Step>

  <Step>
    ### 初始化 Openfort SDK（服务器端）

    在服务器端创建 Openfort 客户端实例：

    ```ts title="openfort.ts"
    import Openfort from "@openfort/openfort-node";

    export const openfortSDK = new Openfort(process.env.OPENFORT_SECRET_KEY);
    ```
  </Step>

  <Step>
    ### 在 Better Auth 配置中添加插件

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth";
    import { bearer } from "better-auth/plugins";
    import { openfort, encryptionSession } from "@openfort/better-auth";
    import { openfortSDK } from "./openfort";

    export const auth = betterAuth({
      database: {
        // 你的数据库配置
      },
      emailAndPassword: {
        enabled: true,
      },
      plugins: [
        bearer(), // 令牌认证必需
        openfort({
          client: openfortSDK,
          use: [
            encryptionSession({
              config: {
                apiKey: process.env.SHIELD_PUBLISHABLE_KEY,
                secretKey: process.env.SHIELD_SECRET_KEY,
                encryptionPart: process.env.SHIELD_ENCRYPTION_SHARE,
              },
            }),
          ],
        }),
      ],
    });
    ```

    <Callout>
      必须先添加 `bearer` 插件，再添加 `openfort` 插件，以启用基于令牌的认证。
    </Callout>
  </Step>

  <Step>
    ### 创建 Better Auth 客户端

    ```ts title="auth-client.ts"
    import { createAuthClient } from 'better-auth/react';
    import { bearer } from "better-auth/plugins/bearer/client";

    export const authClient = createAuthClient({
      baseURL: import.meta.env.VITE_BETTERAUTH_URL,
      plugins: [bearer()],
    });
    ```
  </Step>

  <Step>
    ### 在应用中添加 OpenfortProvider

    `OpenfortProvider` 可独立使用 — 仅当你使用 Wagmi 的 hooks 时，才需要 Wagmi 和 React Query。

    ```tsx title="app/providers.tsx"
    import { OpenfortProvider, ThirdPartyOAuthProvider } from "@openfort/react";
    import { authClient } from "./auth-client";

    export function OpenfortProviders({ children }: { children: React.ReactNode }) {
      return (
        <OpenfortProvider
          publishableKey={import.meta.env.VITE_OPENFORT_PUBLISHABLE_KEY}
          walletConfig={{
            shieldPublishableKey: import.meta.env.VITE_SHIELD_PUBLISHABLE_KEY,
            ethereum: {
              ethereumFeeSponsorshipId: import.meta.env.VITE_FEE_SPONSORSHIP_ID,
            },
            solana: {
              cluster: "devnet", // "mainnet-beta" | "devnet" | "testnet"
            },
            getEncryptionSession: async ({ accessToken }) => {
              const response = await fetch(
                import.meta.env.VITE_BETTERAUTH_URL +
                import.meta.env.VITE_BETTERAUTH_BASE_PATH +
                "/encryption-session",
                {
                  method: "POST",
                  headers: {
                    Authorization: `Bearer ${accessToken}`,
                    "Content-Type": "application/json",
                  },
                }
              );

              if (!response.ok) return null;

              const data = await response.json();
              return data.sessionId;
            },
            connectOnLogin: false,
          }}
          thirdPartyAuth={{
            getAccessToken: async () => {
              const session = await authClient.getSession();
              return session?.data?.session?.token ?? null;
            },
            provider: ThirdPartyOAuthProvider.BETTER_AUTH,
          }}
        >
          {children}
        </OpenfortProvider>
      );
    }
    ```

    <Callout>
      The `getEncryptionSession` callback receives `{ accessToken, otpCode, userId }` — the SDK provides these automatically. Set `connectOnLogin` to `false` if you want to handle wallet creation manually. Include only the chain configs (`ethereum`, `solana`) you need.
    </Callout>
  </Step>

  <Step>
    ### Add Wagmi for EVM interactions

    Wrap your provider tree with `OpenfortWagmiBridge` to enable Wagmi hooks (`useSignMessage`, `useWriteContract`, etc.):

    ```tsx title="app/providers.tsx"
    import { OpenfortProvider, ThirdPartyOAuthProvider } from "@openfort/react";
    import { OpenfortWagmiBridge, getDefaultConfig } from "@openfort/react/wagmi";
    import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
    import { polygonAmoy, sepolia } from "viem/chains";
    import { WagmiProvider, createConfig } from "wagmi";
    import { authClient } from "./auth-client";

    const wagmiConfig = createConfig(
      getDefaultConfig({
        appName: "Your App Name",
        chains: [polygonAmoy, sepolia]
      })
    );

    const queryClient = new QueryClient();

    export function OpenfortProviders({ children }: { children: React.ReactNode }) {
      return (
        <WagmiProvider config={wagmiConfig}>
          <QueryClientProvider client={queryClient}>
            <OpenfortWagmiBridge>
              <OpenfortProvider
                publishableKey={import.meta.env.VITE_OPENFORT_PUBLISHABLE_KEY}
                walletConfig={{
                  shieldPublishableKey: import.meta.env.VITE_SHIELD_PUBLISHABLE_KEY,
                  ethereum: {
                    ethereumFeeSponsorshipId: import.meta.env.VITE_FEE_SPONSORSHIP_ID,
                  },
                  getEncryptionSession: async ({ accessToken }) => {
                    const response = await fetch(
                      import.meta.env.VITE_BETTERAUTH_URL +
                      import.meta.env.VITE_BETTERAUTH_BASE_PATH +
                      "/encryption-session",
                      {
                        method: "POST",
                        headers: {
                          Authorization: `Bearer ${accessToken}`,
                          "Content-Type": "application/json",
                        },
                      }
                    );

                    if (!response.ok) return null;

                    const data = await response.json();
                    return data.sessionId;
                  },
                  connectOnLogin: false,
                }}
                thirdPartyAuth={{
                  getAccessToken: async () => {
                    const session = await authClient.getSession();
                    return session?.data?.session?.token ?? null;
                  },
                  provider: ThirdPartyOAuthProvider.BETTER_AUTH,
                }}
              >
                {children}
              </OpenfortProvider>
            </OpenfortWagmiBridge>
          </QueryClientProvider>
        </WagmiProvider>
      );
    }
    ```

    <Callout>
      `OpenfortWagmiBridge` 和 `getDefaultConfig` 从 `@openfort/react/wagmi` 导入。
    </Callout>
  </Step>

  <Step>
    ### 配置 Openfort 控制面板

    在 [Openfort 控制面板](https://dashboard.openfort.io/)：

    1. 进入 **身份验证提供商（Authentication Providers）**
    2. 添加 **Better Auth** 作为提供商
    3. 设置 **Base URL** 为你的后端端点（本地开发时可使用 ngrok 或类似工具）
    4. （可选）创建用于交易资助的 **策略（Policy）**
    5. （可选）将 **策略 ID（Policy ID）** 复制到环境变量

    <Callout type="warn">
      本地开发请使用 ngrok 等隧道服务将后端暴露给 Openfort。
    </Callout>
  </Step>
</Steps>

## 使用方法

### 钱包创建（EVM）

使用 `useEthereumEmbeddedWallet` hook 创建嵌入式 EVM 钱包：

```tsx
import { useEthereumEmbeddedWallet } from "@openfort/react/ethereum";
import { RecoveryMethod } from "@openfort/react";

export default function CreateWallet() {
  const { status, create, activeWallet } = useEthereumEmbeddedWallet();

  if (status === "creating") {
    return <div>正在创建钱包...</div>;
  }

  if (status === "connected" && activeWallet) {
    return <div>Wallet: {activeWallet.address}</div>;
  }

  return (
    <div>
      <button
        onClick={() => create({ recoveryMethod: RecoveryMethod.PASSKEY })}
      >
        使用 Passkey 创建
      </button>

      <button
        onClick={() => create({ recoveryMethod: RecoveryMethod.AUTOMATIC })}
      >
        使用自动恢复创建
      </button>

      <button
        onClick={() =>
          create({
            recoveryMethod: RecoveryMethod.PASSWORD,
            password: "your-secure-password",
          })
        }
      >
        使用密码创建
      </button>

    </div>
  );
}
```

### 钱包创建（Solana）

使用 `useSolanaEmbeddedWallet` 创建嵌入式 Solana 钱包：

```tsx
import { useSolanaEmbeddedWallet } from "@openfort/react/solana";
import { RecoveryMethod } from "@openfort/react";

export default function CreateSolanaWallet() {
  const { status, create, activeWallet } = useSolanaEmbeddedWallet();

  if (status === "connected" && activeWallet) {
    return <div>Solana 钱包地址: {activeWallet.address}</div>;
  }

  return (
    <button
      onClick={() => create({ recoveryMethod: RecoveryMethod.AUTOMATIC })}
      disabled={status === "creating"}
    >
      {status === "creating" ? "创建中..." : "创建 Solana 钱包"}
    </button>
  );
}
```

<Callout>
  Solana 需要额外安装 `@solana/kit` 依赖，并在 `walletConfig` 中配置 `solana` 参数。
</Callout>

### 签名消息

使用 Wagmi 的 hook 来签名消息：

```tsx
import { useSignMessage } from "wagmi";

export default function SignMessage() {
  const { data, signMessage, error } = useSignMessage();

  return (
    <div>
      <form
        onSubmit={(event) => {
          event.preventDefault();
          const formData = new FormData(event.target as HTMLFormElement);
          signMessage({ message: formData.get("message") as string });
        }}
      >
        <input name="message" placeholder="要签名的消息" />
        <button type="submit">签名消息</button>
      </form>

      {data && <p>签名结果：{data}</p>}
      {error && <p className="text-red-500">{error.message}</p>}
    </div>
  );
}
```

### 执行交易

使用 Wagmi 执行智能合约交易：

```tsx
import { parseAbi } from "viem";
import { useAccount, useWriteContract } from "wagmi";

const CONTRACT_ADDRESS = "0x...";

export default function MintTokens() {
  const { address } = useAccount();
  const { data: hash, isPending, writeContract, error } = useWriteContract();

  const handleMint = (amount: string) => {
    if (!address) return;

    writeContract({
      address: CONTRACT_ADDRESS,
      abi: parseAbi(["function mint(address to, uint256 amount)"]),
      functionName: "mint",
      args: [address, BigInt(amount)],
    });
  };

  return (
    <div>
      <button
        onClick={() => handleMint("1000")}
        disabled={isPending || !address}
      >
        {isPending ? "铸造中..." : "铸造代币"}
      </button>

      {hash && <p>交易哈希：{hash}</p>}
      {error && <p className="text-red-500">{error.message}</p>}
    </div>
  );
}
```

### 访问用户信息

使用 `useUser` 获取用户及钱包信息：

```tsx
import { useUser } from "@openfort/react";

export default function UserProfile() {
  const { user, isAuthenticated, isConnected, linkedAccounts } = useUser();

  if (!isAuthenticated) {
    return <p>未认证</p>;
  }

  return (
    <div>
      <h2>欢迎，{user?.player?.name || linkedAccounts?.[0]?.email}</h2>
      <p>已连接：{isConnected ? "是" : "否"}</p>
    </div>
  );
}
```

## 配置选项

### 服务器端插件参数

```ts
interface OpenfortPluginOptions {
  client: Openfort; // Openfort SDK 实例（来自 @openfort/openfort-node）
  use?: Array<EncryptionSession>; // 加密会话配置
}
```

### 加密会话配置

```ts
interface EncryptionSessionConfig {
  config: {
    apiKey: string; // Shield API 密钥
    secretKey: string; // Shield 秘钥
    encryptionPart: string; // Shield 加密份额
  };
}
```

### 客户端 OpenfortProvider 属性

```ts
interface OpenfortProviderProps {
  publishableKey: string; // Openfort 发布钥匙
  walletConfig?: {
    shieldPublishableKey: string; // Shield 发布钥匙
    connectOnLogin?: boolean; // 认证后自动恢复/创建钱包（默认：true）
    passkeyDisplayName?: string; // Passkey 提示显示名
    // 加密会话（选其一）：
    getEncryptionSession?: (params: {
      accessToken: string;
      otpCode?: string;
      userId: string;
    }) => Promise<string>;
    createEncryptedSessionEndpoint?: string; // 或提供 API 端点
    // 链相关配置：
    ethereum?: EthereumConfig; // EVM 链配置（chainId, rpcUrls, policyId）
    solana?: SolanaConfig; // Solana 集群和 RPC 配置
  };
  thirdPartyAuth?: {
    getAccessToken: () => Promise<string | null>; // 获取 Better Auth 令牌
    provider: ThirdPartyOAuthProvider; // 认证提供商类型
  };
  debugMode?: boolean; // 开启调试日志
  children: React.ReactNode;
}
```

## 高级用法

### 恢复方式

Openfort 支持三种钱包恢复方式：

- **自动恢复**：基于 Shield 加密的恢复（无需用户交互）  
- **Passkey**：生物认证（Face ID，Touch ID，Windows Hello）  
- **密码**：用户自定义密码恢复  

```tsx
import { useEthereumEmbeddedWallet } from "@openfort/react/ethereum";
import { RecoveryMethod } from "@openfort/react";

const { create } = useEthereumEmbeddedWallet();

// Passkey 恢复
create({ recoveryMethod: RecoveryMethod.PASSKEY });

// 自动恢复
create({ recoveryMethod: RecoveryMethod.AUTOMATIC });

// 密码恢复
create({
  recoveryMethod: RecoveryMethod.PASSWORD,
  password: "your-secure-password",
});
```

### 读取合约数据

使用 Wagmi 的 `useReadContract` hook 从智能合约读取数据：

```tsx
import { useReadContract } from "wagmi";

export default function TokenBalance() {
  const { data: balance, isLoading, error } = useReadContract({
    address: "0x...",
    abi: [
      {
        type: "function",
        name: "balanceOf",
        stateMutability: "view",
        inputs: [{ name: "account", type: "address" }],
        outputs: [{ type: "uint256" }],
      },
    ],
    functionName: "balanceOf",
    args: ["0x..."],
  });

  if (isLoading) return <div>加载中...</div>;
  if (error) return <div>错误：{error.message}</div>;

  return <div>余额：{balance?.toString()}</div>;
}
```

## 故障排查

### 加密会话错误

若获取加密会话报错：

1. 确认 Bearer 插件已在 Better Auth 配置中启用  
2. 确认 `SHIELD_PUBLISHABLE_KEY`、`SHIELD_SECRET_KEY` 和 `SHIELD_ENCRYPTION_SHARE` 配置正确  
3. 调用 `getEncryptionSession()` 前确认 Better Auth 会话有效  

### CORS 问题

若遇到 CORS 错误：

1. 将前端 URL 添加到 Better Auth 配置中的 `trustedOrigins` 中：

```ts
export const auth = betterAuth({
  trustedOrigins: [process.env.FRONTEND_URL],
  // ...
});
```

2. 确保 Express 服务器配置了 CORS：

```ts
import cors from "cors";

app.use(
  cors({
    origin: process.env.FRONTEND_URL,
    credentials: true,
  })
);
```

### 身份验证提供商配置

若 Openfort 无法验证令牌：

1. 确认 Openfort 控制面板的 **Base URL** 与你的后端一致  
2. 本地开发时使用隧道服务（ngrok、localtunnel）  
3. 确认 `/api/auth/encryption-session` 接口可访问。`encryption-session` 路径由 Openfort Better Auth 插件添加
