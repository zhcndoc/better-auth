---
title: API 密钥
description: Better Auth 的 API 密钥插件。
---

API 密钥插件允许您为应用程序创建和管理 API 密钥。它通过验证 API 密钥来认证和授权 API 请求。

## 功能

- 创建、管理和验证 API 密钥
- [内置速率限制](/docs/plugins/api-key/advanced#rate-limiting)
- [自定义过期时间、剩余次数及补充系统](/docs/plugins/api-key/advanced#remaining-refill-and-expiration)
- [API 密钥元数据](/docs/plugins/api-key/advanced#metadata)
- 自定义前缀
- [基于 API 密钥的会话](/docs/plugins/api-key/advanced#sessions-from-api-keys)
- [支持二级存储](/docs/plugins/api-key/advanced#storage-modes) 实现高性能的 API 密钥查找
- [多配置支持](/docs/plugins/api-key/advanced#multiple-configurations) 以适配不同类型的 API 密钥
- 除用户拥有的密钥外，还支持[组织拥有的 API 密钥](/docs/plugins/api-key/advanced#organization-owned-api-keys)

## 安装

<Steps>
    <Step>
        ### 安装插件

        ```package-install
        @better-auth/api-key
        ```
    </Step>
    <Step>
        ### 将插件添加到服务器

        ```ts title="auth.ts"
        import { betterAuth } from "better-auth"
        import { apiKey } from "@better-auth/api-key" // [!code highlight]

        export const auth = betterAuth({
            plugins: [
                apiKey() // [!code highlight]
            ]
        })
        ```
    </Step>
    <Step>
        ### 迁移数据库

        运行迁移或生成模式(schema)以向数据库添加必要的字段和表。

        <Tabs items={["migrate", "generate"]}>
            <Tab value="migrate">
            ```package-install
            npx auth migrate
            ```
            </Tab>
            <Tab value="generate">
            ```package-install
            npx auth generate
            ```
            </Tab>
        </Tabs>
        详细信息见[模式(Schema)](/docs/plugins/api-key/reference#schema)部分，可手动添加字段。
    </Step>
    <Step>
        ### 添加客户端插件

        ```ts title="auth-client.ts"
        import { createAuthClient } from "better-auth/client"
        import { apiKeyClient } from "@better-auth/api-key/client"  // [!code highlight]

        export const authClient = createAuthClient({
            plugins: [
                apiKeyClient() // [!code highlight]
            ]
        })
        ```
    </Step>

</Steps>

## 使用方法

您可以在[这里](/docs/plugins/api-key/reference#api-key-plugin-options)查看 API 密钥插件的所有选项。

### 创建 API 密钥

<APIMethod
  path="/api-key/create"
  method="POST"
  serverOnlyNote="如果在服务器创建 API 密钥且无法访问请求头，必须传入 `userId` 属性（适用于用户拥有的密钥）或 `organizationId`（适用于组织拥有的密钥）。"
  clientOnlyNote="您可以使用服务器方法来调整更具体的 API 密钥配置。"
>
```ts
type createApiKey = {
    /**
     * 要使用的配置 ID。如果未提供，使用默认配置。
     */
    configId?: string
    /**
     * API 密钥名称。
     */
    name?: string = 'project-api-key'
    /**
     * API 密钥的过期时间（秒）。
     */
    expiresIn?: number = 60 * 60 * 24 * 7
    /**
     * API 密钥所属用户 ID，仅服务器使用。
     * 不使用会话头时，用户拥有的密钥必须设置此项。
     * @serverOnly
     */
    userId?: string = "user-id"
    /**
     * API 密钥所属组织 ID。
     * 针对组织拥有的密钥（配置中设置 `references: "organization"`）必填。
     */
    organizationId?: string = "org-id"
    /**
     * API 密钥的前缀。
     */
    prefix?: string = 'project-api-key'
    /**
     * 剩余请求次数，仅服务器使用。
     * @serverOnly
     */
    remaining?: number = 100
    /**
     * API 密钥的元数据。
     */
    metadata?: any | null = { someKey: 'someValue' }
    /**
     * 剩余请求次数的补充数量，仅服务器使用。
     * @serverOnly
     */
    refillAmount?: number = 100
    /**
     * 补充请求次数的间隔时间（毫秒），仅服务器使用。
     * @serverOnly
     */
    refillInterval?: number = 1000
    /**
     * 计数请求的时间窗口（毫秒）。达到最大请求数后，请求将被拒绝，直到时间窗口结束并重置，服务器端使用。
     * @serverOnly
     */
    rateLimitTimeWindow?: number = 1000
    /**
     * 时间窗口内允许的最大请求数。达到后请求被拒，等待时间窗口重置，服务器端使用。
     * @serverOnly
     */
    rateLimitMax?: number = 100
    /**
     * 是否启用速率限制，仅服务器使用。
     * @serverOnly
     */
    rateLimitEnabled?: boolean = true
    /**
     * API 密钥权限。
     * @serverOnly
     */
    permissions?: Record<string, string[]>
}
```
</APIMethod>

<Callout>API 密钥可以是用户拥有，也可以是组织拥有，具体取决于配置中的 `references` 设置。</Callout>

#### 返回结果

返回包含 `key` 值的 `ApiKey` 对象供您使用。
如果发生异常，则抛出 `APIError`。

### 验证 API 密钥

<APIMethod
  path="/api-key/verify"
  method="POST"
  isServerOnly
>
```ts
const permissions = { // 可选：需检查的权限。
  projects: ["read", "read-write"],
}

type verifyApiKey = {
    /**
     * 用于验证的配置 ID。如果未提供，使用默认配置。
     */
    configId?: string
    /**
     * 需要验证的密钥。
     */
    key: string = "your_api_key_here"
    /**
     * 需要验证的权限，可选。
     */
    permissions?: Record<string, string[]>
}
```
</APIMethod>


#### 返回结果

```ts
type Result = {
  valid: boolean;
  error: { message: string; code: string } | null;
  key: Omit<ApiKey, "key"> | null;
};
```

### 获取 API 密钥

<APIMethod
  path="/api-key/get"
  method="GET"
  requireSession
>
```ts
type getApiKey = {
    /**
     * 用于查找 API 密钥的配置 ID。如果未提供，使用默认配置。
     */
    configId?: string
    /**
     * API 密钥的 ID。
     */
    id: string = "some-api-key-id"
}
```
</APIMethod>

#### 返回结果

返回除 `key` 字段外的所有 API 密钥详细信息。
如果失败，会抛出 `APIError`。

```ts
type Result = Omit<ApiKey, "key">;
```

### 更新 API 密钥

<APIMethod path="/api-key/update" method="POST">
```ts
type updateApiKey = {
    /**
     * 用于查找 API 密钥的配置 ID。如果未提供，使用默认配置。
     */
    configId?: string
    /**
     * 待更新 API 密钥的 ID。
     */
    keyId: string = "some-api-key-id"
    /**
     * API 密钥所属用户 ID，仅服务器使用。
     * @serverOnly
     */
    userId?: string = "some-user-id"
    /**
     * 密钥名称。
     */
    name?: string = "some-api-key-name"
    /**
     * 是否启用 API 密钥，仅服务器使用。
     * @serverOnly
     */
    enabled?: boolean = true
    /**
     * 剩余请求次数，仅服务器使用。
     * @serverOnly
     */
    remaining?: number = 100
    /**
     * 补充数量，仅服务器使用。
     * @serverOnly
     */
    refillAmount?: number = 100
    /**
     * 补充时间间隔（毫秒），仅服务器使用。
     * @serverOnly
     */
    refillInterval?: number = 1000
    /**
     * API 密钥元数据，仅服务器使用。
     * @serverOnly
     */
    metadata?: any | null = { "key": "value" }
    /**
     * 过期时间（秒），仅服务器使用。
     * @serverOnly
     */
    expiresIn?: number = 60 * 60 * 24 * 7
    /**
     * 是否启用速率限制，仅服务器使用。
     * @serverOnly
     */
    rateLimitEnabled?: boolean = true
    /**
     * 计数请求的时间窗口（毫秒），仅服务器使用。
     * @serverOnly
     */
    rateLimitTimeWindow?: number = 1000
    /**
     * 时间窗口内允许的最大请求数，服务器端使用。
     * @serverOnly
     */
    rateLimitMax?: number = 100
    /**
     * 更新 API 密钥的权限，仅服务器使用。
     * @serverOnly
     */
    permissions?: Record<string, string[]>
}
```
</APIMethod>

#### 返回结果

如果失败，则抛出 `APIError`。
成功时返回 API 密钥详细信息，但不包含 `key` 字段。

### 删除 API 密钥

<APIMethod
  path="/api-key/delete"
  method="POST"
  requireSession

### 列出 API 密钥

<APIMethod
  path="/api-key/list"
  method="GET"
  requireSession
>
```ts
type listApiKeys = {
    /**
     * 按配置 ID 过滤。如果未提供，返回所有配置下的密钥。
     */
    configId?: string
    /**
     * 组织 ID。提供则返回该组织拥有的密钥。
     * 未提供则返回当前会话用户拥有的密钥。
     */
    organizationId?: string
    /**
     * 返回的 API 密钥数量限制。
     */
    limit?: number
    /**
     * 偏移量，用于分页。
     */
    offset?: number
    /**
     * 排序字段（如 "createdAt", "name", "expiresAt"）。
     */
    sortBy?: string
    /**
     * 排序方向。
     */
    sortDirection?: "asc" | "desc"
}
```
</APIMethod>

#### 返回结果

如果失败，抛出 `APIError`。
成功时返回分页响应：

```ts
type Result = {
  apiKeys: Omit<ApiKey, "key">[];
  total: number;
  limit?: number;
  offset?: number;
};
```

#### 分页用例

```ts
// 获取当前用户的前 10 个 API 密钥
const result = await authClient.apiKey.list({
  query: { limit: 10 }
});

// 获取第二页（每页10条）
const page2 = await authClient.apiKey.list({
  query: { limit: 10, offset: 10 }
});

// 按创建时间排序（最新优先）
const sorted = await authClient.apiKey.list({
  query: { sortBy: "createdAt", sortDirection: "desc" }
});

// 分页和排序结合
const combined = await authClient.apiKey.list({
  query: { 
    limit: 20, 
    offset: 0, 
    sortBy: "name", 
    sortDirection: "asc" 
  }
});

// 列出组织拥有的密钥
const orgKeys = await authClient.apiKey.list({
  query: { organizationId: "org_123" }
});

// 列出指定配置的组织密钥
const orgPublicKeys = await authClient.apiKey.list({
  query: { 
    organizationId: "org_123",
    configId: "public" 
  }
});
```

此功能会删除所有已过期的 API 密钥。

<APIMethod
  path="/api-key/delete-all-expired-api-keys"
  method="POST"
  isServerOnly
>
```ts
type deleteAllExpiredApiKeys = {
}
```
</APIMethod>

<Callout>
  我们会在每次调用任何 apiKey 插件的接口时自动删除过期密钥，
  但调用频率限制为每 10 秒冷却一次，以防多次访问数据库。
</Callout>
