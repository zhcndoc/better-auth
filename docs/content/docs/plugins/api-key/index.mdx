---
title: API 密钥
description: Better Auth 的 API 密钥插件。
---

API 密钥插件允许您为应用程序创建和管理 API 密钥。它通过验证 API 密钥来认证和授权 API 请求。

## 功能

* 创建、管理和验证 API 密钥
* [内置的速率限制](/docs/plugins/api-key/advanced#rate-limiting)
* [自定义过期时间、剩余计数和补充系统](/docs/plugins/api-key/advanced#remaining-refill-and-expiration)
* [API 密钥的元数据](/docs/plugins/api-key/advanced#metadata)
* 自定义前缀
* [来自 API 密钥的会话](/docs/plugins/api-key/advanced#sessions-from-api-keys)
* [二级存储支持](/docs/plugins/api-key/advanced#storage-modes) 用于高性能的 API 密钥查找
* [多个配置](/docs/plugins/api-key/advanced#multiple-configurations) 用于不同类型的 API 密钥
* [组织拥有的 API 密钥](/docs/plugins/api-key/advanced#organization-owned-api-keys)，以及用户拥有的密钥

## 安装

<Steps>
  <Step>
    ### 安装插件

    ```package-install
    @better-auth/api-key
    ```
  </Step>

  <Step>
    ### 将插件添加到服务器

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth"
    import { apiKey } from "@better-auth/api-key" // [!code highlight]

    export const auth = betterAuth({
        plugins: [
            apiKey() // [!code highlight]
        ]
    })
    ```
  </Step>

  <Step>
    ### 迁移数据库

    运行迁移或生成架构以向数据库添加必要的字段和表。

    <Tabs items={["migrate", "generate"]}>
      <Tab value="migrate">
        ```package-install
        npx auth migrate
        ```
      </Tab>

      <Tab value="generate">
        ```package-install
        npx auth generate
        ```
      </Tab>
    </Tabs>

    请参阅 [架构](/docs/plugins/api-key/reference#schema) 部分以手动添加字段。
  </Step>

  <Step>
    ### 添加客户端插件

    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client"
    import { apiKeyClient } from "@better-auth/api-key/client"  // [!code highlight]

    export const authClient = createAuthClient({
        plugins: [
            apiKeyClient() // [!code highlight]
        ]
    })
    ```
  </Step>
</Steps>

## 使用方法

您可以在[这里](/docs/plugins/api-key/reference#api-key-plugin-options)查看 API 密钥插件的所有选项。

### 创建 API 密钥

<APIMethod path="/api-key/create" method="POST" serverOnlyNote="如果在服务器上创建 API 密钥且无法访问标头，则必须传递 `userId` 属性（用于用户拥有的密钥）或 `organizationId`（用于组织拥有的密钥）。" clientOnlyNote="您可以使用服务器方法来调整更具体的 API 密钥配置。">
  ```ts
  type createApiKey = {
      /**
       * 要使用的配置 ID。如果未提供，则使用默认配置。
       */
      configId?: string
      /**
       * API 密钥的名称。
       */
      name?: string = 'project-api-key'
      /**
       * API 密钥的过期时间（以秒为单位）。
       */
      expiresIn?: number = 60 * 60 * 24 * 7
      /**
       * 拥有该 API 密钥的用户 ID。server-only。
       * 当不使用会话标头时，这是用户拥有的密钥所必需的。
       * @serverOnly
       */
      userId?: string = "user-id"
      /**
       * 拥有该 API 密钥的组织 ID。
       * 对于 `references: "organization"` 的配置是必需的。
       */
      organizationId?: string = "org-id"
      /**
       * API 密钥的前缀。
       */
      prefix?: string = 'project-api-key'
      /**
       * 剩余的请求数量。server-only。
       * @serverOnly
       */
      remaining?: number = 100
      /**
       * API 密钥的元数据。
       */
      metadata?: any | null = { someKey: 'someValue' }
      /**
       * 要补充的剩余计数。server-only。
       * @serverOnly
       */
      refillAmount?: number = 100
      /**
       * 补充 API 密钥的间隔（以毫秒为单位）。server-only。
       * @serverOnly
       */
      refillInterval?: number = 1000
      /**
       * 每个请求计数的时间窗口（以毫秒为单位）。一旦达到 `maxRequests`，请求将被拒绝，直到 `timeWindow` 过去，之后 `timeWindow` 将重置。server-only。
       * @serverOnly
       */
      rateLimitTimeWindow?: number = 1000
      /**
       * 在一个窗口内允许的最大请求数。一旦达到 `maxRequests`，请求将被拒绝，直到 `timeWindow` 过去，之后 `timeWindow` 将重置。server-only。
       * @serverOnly
       */
      rateLimitMax?: number = 100
      /**
       * 是否启用速率限制。server-only。
       * @serverOnly
       */
      rateLimitEnabled?: boolean = true
      /**
       * API 密钥的权限。
       * @serverOnly
       */
      permissions?: Record<string, string[]>
  }
  ```
</APIMethod>

<Callout>API 密钥可以是用户拥有的，也可以是组织拥有的，具体取决于配置中的 `references` 设置。</Callout>

#### 返回结果

返回包含 `key` 值的 `ApiKey` 对象供您使用。
如果发生异常，则抛出 `APIError`。

***

### 验证 API 密钥

<APIMethod path="/api-key/verify" method="POST" isServerOnly>
  ```ts
  const permissions = { // 需要检查的权限是可选的
    projects: ["read", "read-write"],
  }

  type verifyApiKey = {
      /**
       * 用于验证的配置 ID。如果未提供，则使用默认配置。
       */
      configId?: string
      /**
       * 要验证的密钥。
       */
      key: string = "your_api_key_here"
      /**
       * 要验证的权限。可选。
       */
      permissions?: Record<string, string[]>
  }
  ```
</APIMethod>

#### 结果

```ts
type Result = {
  valid: boolean;
  error: { message: string; code: string } | null;
  key: Omit<ApiKey, "key"> | null;
};
```

### 获取 API 密钥

<APIMethod path="/api-key/get" method="GET" requireSession>
  ```ts
  type getApiKey = {
      /**
       * 用于 API 密钥查找的配置 ID。如果未提供，则使用默认配置。
       */
      configId?: string
      /**
       * API 密钥的 ID。
       */
      id: string = "some-api-key-id"
  }
  ```
</APIMethod>

#### 返回结果

返回除 `key` 字段外的所有 API 密钥详细信息。
如果失败，则抛出 `APIError`。

```ts
type Result = Omit<ApiKey, "key">;
```

       */

       * 剩余的请求数量。server-only。
       * @serverOnly
       */
      remaining?: number = 100
      /**
       * 补充数量。server-only。
       * @serverOnly
       */
      refillAmount?: number = 100
      /**
       * 补充间隔（以毫秒为单位）。server-only。

       */

       * API 密钥的元数据。server-only。
       * @serverOnly
       */

      /**
       * API 密钥的过期时间（以秒为单位）。server-only。
       * @serverOnly
       */
      expiresIn?: number = 60 * 60 * 24 * 7
      /**
       * 是否启用速率限制。server-only。
       * @serverOnly
       */
      rateLimitEnabled?: boolean = true
      /**
       * 每个请求计数的时间窗口（以毫秒为单位）。server-only。
       * @serverOnly
       */
      rateLimitTimeWindow?: number = 1000
      /**
       * 在一个窗口内允许的最大请求数。一旦达到 `maxRequests`，请求将被拒绝，直到 `timeWindow` 过去，之后 `timeWindow` 将重置。server-only。
       * @serverOnly
       */
      rateLimitMax?: number = 100
      /**
       * 更新 API 密钥的权限。server-only。

### 删除 API 密钥

<APIMethod path="/api-key/delete" method="POST" requireSession note="此端点从用户角度尝试删除 API 密钥。它会检查用户 ID 是否与密钥所有者匹配，以确定是否可以删除它。如果您希望在不进行这些检查的情况下删除密钥，我们建议您使用 ORM 直接修改数据库。">
  ```ts
  type deleteApiKey = {
      /**
       * 用于 API 密钥查找的配置 ID。如果未提供，则使用默认配置。
       */
      configId?: string
      /**
       * 要删除的 API 密钥的 ID。
       */
      keyId: string = "some-api-key-id"
  }
  ```
</APIMethod>

#### 返回结果

如果失败，则抛出 `APIError`。
成功时返回：

```ts
type Result = {
  success: boolean;
};
```

### 列出 API 密钥

<APIMethod path="/api-key/list" method="GET" requireSession>
  ```ts
  type listApiKeys = {
      /**
       * 根据配置 ID 进行筛选。如果未提供，则返回所有配置的密钥。
       */
      configId?: string
      /**
       * 要列出密钥的组织 ID。如果提供，则返回组织拥有的密钥。
       * 如果未提供，则返回当前会话用户拥有的密钥。
       */
      organizationId?: string
      /**
       * 返回的 API 密钥数量。
       */
      limit?: number
      /**
       * 分页开始位置（偏移量）。
       */
      offset?: number
      /**
       * 排序字段（例如："createdAt"、"name"、"expiresAt"）。
       */
      sortBy?: string
      /**
       * 排序方向。
       */
      sortDirection?: "asc" | "desc"
  }
  ```
</APIMethod>

#### 返回结果

如果失败，抛出 `APIError`。
成功时返回分页响应：

```ts
type Result = {
  apiKeys: Omit<ApiKey, "key">[];
  total: number;
  limit?: number;
  offset?: number;
};
```

#### 分页用例

```ts
// 获取当前用户的前 10 个 API 密钥
const result = await authClient.apiKey.list({
  query: { limit: 10 }
});

// 获取第二页（每页10条）
const page2 = await authClient.apiKey.list({
  query: { limit: 10, offset: 10 }
});

// 按创建时间排序（最新优先）
const sorted = await authClient.apiKey.list({
  query: { sortBy: "createdAt", sortDirection: "desc" }
});

// 分页和排序结合
const combined = await authClient.apiKey.list({
  query: { 
    limit: 20, 
    offset: 0, 
    sortBy: "name", 
    sortDirection: "asc" 
  }
});

// 列出组织拥有的密钥
const orgKeys = await authClient.apiKey.list({
  query: { organizationId: "org_123" }
});

// 列出指定配置的组织密钥
const orgPublicKeys = await authClient.apiKey.list({
  query: { 
    organizationId: "org_123",
    configId: "public" 
  }
});
```

  type deleteAllExpiredApiKeys = {
  }
