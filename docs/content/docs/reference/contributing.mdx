感谢你对为 Better Auth 贡献代码的兴趣！本指南是为 Better Auth 贡献代码的简明指南。

## 入门

在开始之前，这里有一些重要资源：

* 查看我们的现有 <Link href="https://github.com/better-auth/better-auth/issues">问题</Link> 和 <Link href="https://github.com/better-auth/better-auth/pulls">拉取请求</Link>
* 在 <Link href="https://discord.gg/better-auth">Discord</Link> 中参与社区讨论

开始开发前：

<Callout type="warn">
  请确保你已安装 <Link href="https://nodejs.org/en/download">Node.JS</Link>，建议使用 LTS 版本。
</Callout>

<Steps>
  <Step>
    ### 1. Fork 仓库

    访问 [https://github.com/better-auth/better-auth](https://github.com/better-auth/better-auth)

    点击右上角“Fork”按钮。
  </Step>

  <Step>
    ### 2. 克隆你的 Fork

    ```bash
    # 将 YOUR-USERNAME 替换为你的 GitHub 用户名
    git clone https://github.com/YOUR-USERNAME/better-auth.git
    cd better-auth
    ```
  </Step>

  <Step>
    ### 3. 安装依赖

    确保已安装 <Link href="https://pnpm.io/installation">pnpm</Link>！

    ```bash
    pnpm install
    ```
  </Step>

  <Step>
    ### 4. 准备环境文件

    复制示例环境文件以创建新的 `.env` 文件。

    ```bash
    cp -n ./docs/.env.example ./docs/.env
    ```
  </Step>
</Steps>

## 进行更改

当你确定想要贡献的内容后，就可以开始更改了。以下是入门步骤：

<Steps>
  <Step>
    ### 1. 创建新分支

    ```bash
    # 添加上游远程仓库（如果尚未添加）
    git remote add upstream https://github.com/better-auth/better-auth.git

    # 确保你在 main 分支
    git checkout main

    # 拉取最新更改
    git pull upstream main

    # 创建并切换到新分支
    git checkout -b feature/your-feature-name
    ```
  </Step>

  <Step>
    ### 2. 启动开发服务器

    启动开发服务器：

    ```bash
    pnpm dev
    ```

    启动文档服务器：

    ```bash
    pnpm -F docs dev
    ```
  </Step>

  <Step>
    ### 3. 进行更改

    * 对代码库进行更改。

    * 如有需要，编写测试。（阅读有关测试的更多信息 <Link href="/docs/reference/contributing#testing">这里</Link>）

    * 更新文档。（阅读有关文档的更多信息 <Link href="/docs/reference/contributing#documentation">这里</Link>）
  </Step>
</Steps>

```ts

import { getTestInstance } from "./test-utils/test-instance";

describe("Feature", () => {
    it("should work as expected", async () => {
        const { client } = await getTestInstance();
        // 在这里编写测试代码

    });

```

### 使用测试实例辅助工具

测试实例辅助工具现已支持改进的异步上下文管理以处理用户会话：

```ts
const { client, runWithUser, signInWithTestUser } = await getTestInstance();

// 以指定用户上下文运行测试
await runWithUser("user@example.com", "password", async (headers) => {
    // 该代码块内所有客户端调用将使用该用户的会话
    const response = await client.getSession();
    // headers 会自动应用
});

// 或使用测试用户的异步上下文
const { runWithDefaultUser } = await signInWithTestUser();
await runWithDefaultUser(async (headers) => {
    // 此处代码在测试用户会话上下文中运行
});
```

### 测试最佳实践

* 遵循我们的编码标准
* 保持拉取请求专注于单一更改
