## 开发环境搭建

开始开发前：

<Callout type="warn">
  请确保你已安装 <Link href="https://nodejs.org/en/download">Node.JS</Link> ，建议使用 LTS 版本。
</Callout>

<Steps>
  <Step>
    ### 1. Fork 仓库

    访问 [https://github.com/better-auth/better-auth](https://github.com/better-auth/better-auth)

    点击右上角的 “Fork” 按钮。
  </Step>

  <Step>
    ### 2. 克隆你的 Fork

    ```bash
    # 用你的 GitHub 用户名替换 YOUR-USERNAME
    git clone https://github.com/YOUR-USERNAME/better-auth.git
    cd better-auth
    ```
  </Step>

  <Step>
    ### 3. 安装依赖

    请确保你已经安装了 <Link href="https://pnpm.io/installation">pnpm</Link>！

    ```bash
    pnpm install
    ```
  </Step>

  <Step>
    ### 4. 准备环境变量文件

    复制示例 env 文件创建新的 `.env` 文件。

    ```bash
    cp -n ./docs/.env.example ./docs/.env
    ```
  </Step>
</Steps>

## 进行更改

当你确定想要贡献的内容后，就可以开始更改了。以下是入门步骤：

<Steps>
  <Step>
    ### 1. 创建新分支

    ```bash
    # 添加上游远程（如果未添加）
    git remote add upstream https://github.com/better-auth/better-auth.git

    # 确保你在 main 分支上
    git checkout main

    # 拉取最新更改
    git pull upstream main

    # 创建并切换到新分支
    git checkout -b feature/your-feature-name
    ```
  </Step>

  <Step>
    ### 2. 启动开发服务器

    启动开发服务器：

    ```bash
    pnpm dev
    ```

    启动文档服务器：

    ```bash
    pnpm -F docs dev
    ```
  </Step>

  <Step>
    ### 3. 进行更改

    * 修改代码库。

    * 如需要，编写测试。（关于测试的更多内容请见 <Link href="/docs/reference/contributing#testing">这里</Link>）

    * 更新文档。（关于文档的更多内容请见 <Link href="/docs/reference/contributing#documentation">这里</Link>）
  </Step>
</Steps>

### 问题和 Bug 修复

* 查看我们的 [GitHub issues](https://github.com/better-auth/better-auth/issues) 中标注为 `good first issue` 的任务
* 报告 Bug 时请包含复现步骤和预期行为
* 在你希望参与的问题上留言，以避免重复工作

### 框架集成

欢迎为更多框架贡献支持：

* 尽可能关注框架无关的解决方案
* 保持集成方案简洁且易于维护
* 所有集成目前都在主包中

### 插件开发

* 核心插件：请先开 issue 讨论你的想法
* 社区插件：可独立开发
* 遵循我们的插件架构指南

### 文档

* 修正拼写和错误
* 添加示例，澄清现有内容
* 确保文档与代码更改同步更新

## 测试

我们使用 Vitest 作为测试工具。请将测试文件放置在对应源码文件旁：

```ts
import { describe, it, expect } from "vitest";
import { getTestInstance } from "./test-utils/test-instance";

describe("Feature", () => {
    it("should work as expected", async () => {
        const { client } = await getTestInstance();
        // 这里写测试代码
        expect(result).toBeDefined();
    });
});
```

### 使用测试实例辅助工具

测试实例辅助工具现已支持改进的异步上下文管理以处理用户会话：

```ts
const { client, runWithUser, signInWithTestUser } = await getTestInstance();

// 以指定用户上下文运行测试
await runWithUser("user@example.com", "password", async (headers) => {
    // 该代码块内所有客户端调用将使用该用户的会话
    const response = await client.getSession();
    // headers 会自动应用
});

// 或使用测试用户的异步上下文
const { runWithDefaultUser } = await signInWithTestUser();
await runWithDefaultUser(async (headers) => {
    // 此处代码在测试用户会话上下文中运行
});
```

### 测试最佳实践

* 编写清晰的提交信息
* 更新文档以反映更改
* 为新功能添加测试
* 遵循我们的编码规范
* 保持拉取请求专注于单一更改
