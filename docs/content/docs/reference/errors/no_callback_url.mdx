---
title: 无回调 URL
description: 请求中未找到回调 URL。
---

## 这是什么？

当请求到达你的 `/api/auth/callback` 端点时，如果 `state` 数据中不包含回调 URL，就会发生此错误。
Better Auth 在流程开始时会将元数据存储在 `state` 中，包括成功登录/关联后要重定向的位置。如果回调时缺少该 URL，我们就无法安全地继续。

## 常见原因

* OAuth 流程不是通过 Better Auth API 启动的，因此 `state` 载荷中从未包含回调 URL。
* 反向代理、CDN 或中间件更改了流程，导致应用读取了不同或为空的 `state`。

## 解决方法

### 通过 Better Auth 启动流程

* 始终使用 Better Auth 内置的方法启动 OAuth，这样 `state` 会生成所需字段。
