title: 远程测量
description: Better Auth 现收集有关一般使用情况的匿名远程测量数据。
---

Better Auth 会收集匿名的使用数据，帮助我们改进项目。这是可选的、透明的，且默认情况下关闭。

## 为什么要收集远程测量？

从 v1.3.5 版本开始，如果启用，Better Auth 会收集有关一般使用情况的匿名远程测量数据。

远程测量数据帮助我们了解 Better Auth 在不同环境中的使用情况，以便提升性能、优先开发功能和更有效地修复问题。它指导我们在性能优化、功能开发和修复漏洞上的决策。所有数据的收集都是完全匿名的，且以隐私为重，用户可随时选择退出。我们努力保持收集内容的透明度。

## 收集了什么数据？

可能报告以下数据点。所有数据均为匿名，仅用于汇总分析。

* **匿名标识符**：来自你的项目（`package.json` 的 name 字段和可选的 `baseURL`）的不可逆哈希。这样我们可以在不知晓你身份的情况下，去重每个项目的事件。
* **运行时**：`{ name: "node" | "bun" | "deno", version }`。
* **环境**：`development`、`production`、`test` 或 `ci` 之一。
* **框架（若探测到）**：像 Next.js、Nuxt、React Router、Astro、SvelteKit 等的 `{ name, version }`。
* **数据库（若探测到）**：如 PostgreSQL、MySQL、SQLite、Prisma、Drizzle、MongoDB 等集成的 `{ name, version }`。
* **系统信息**：平台、操作系统版本、架构、CPU 数量/型号/频率、总内存，以及如 `isDocker`、`isWSL`、`isTTY` 等标识。
* **包管理器**：来自 npm 用户代理的 `{ name, version }`。
* **脱敏的身份验证配置快照**：通过 `getTelemetryAuthConfig` 生成的精简并保护隐私的 `betterAuth` 配置视图。

我们还收集来自 CLI 的匿名远程测量：

* **CLI 生成（`cli_generate`）**：操作结果 `generated | overwritten | appended | no_changes | aborted` 加上脱敏配置。
* **CLI 迁移（`cli_migrate`）**：操作结果 `migrated | no_changes | aborted | unsupported_adapter`，相关时还包括适配器 ID 及脱敏配置。

<Callout type="info">
  你可以通过设置环境变量 `BETTER_AUTH_TELEMETRY_DEBUG=1` 本地审计远程测量，或在身份验证配置中设置 `telemetry: { debug: true }`。在此调试模式下，远程测量事件仅记录到控制台。

  ```ts title="auth.ts"
  export const auth = betterAuth({
    // [!code highlight]
    telemetry: { // [!code highlight]
      debug: true // [!code highlight]
    } // [!code highlight]
  });
  ```
</Callout>

## 我的数据如何受到保护？

所有收集的数据完全匿名，仅在汇总时有用。无法追溯到任何个人来源，仅由少数 Better Auth 核心维护者访问，用于指导产品路线图决策。

* **不收集个人身份信息或密钥**：我们不会收集电子邮件、用户名、令牌、密钥、客户端 ID、客户端密钥或数据库 URL。
* **不发送完整配置**：我们从不发送你完整的 `betterAuth` 配置。相反，我们发送的是一个经过精简和脱敏的快照，仅包含非敏感的开关和计数。
* **按设计进行脱敏**：请参阅 Better Auth 源码中的 [detect-auth-config.ts](https://github.com/better-auth/better-auth/blob/main/packages/telemetry/src/detectors/detect-auth-config.ts)，了解所包含内容的精确结构。它会有意将敏感值转换为布尔值、计数或通用标识符。

## 如何启用？

你可以在身份验证配置中启用远程测量，或设置环境变量。

* 通过身份验证配置启用。

  ```ts title="auth.ts"
  export const auth = betterAuth({
    // [!code highlight]
    telemetry: { // [!code highlight]
      enabled: true // [!code highlight]
    } // [!code highlight]
  });
  ```

* 通过环境变量启用。

  ```txt title=".env"
  # 启用远程测量
  BETTER_AUTH_TELEMETRY=1

  # 禁用远程测量
  BETTER_AUTH_TELEMETRY=0
  ```

### 何时发送远程测量数据？

* 在 `betterAuth` 初始化时（`type: "init"`）。
* 在 CLI 操作时：如上文所述的 `generate` 和 `migrate`。

默认测试环境（`NODE_ENV=test`）下自动禁用远程测量，除非内部工具显式覆盖。
