---
title: Nitro 集成
description: 将 Better Auth 集成到 Nitro。
---

Better Auth 可以集成到你的 [Nitro 应用程序](https://nitro.build/)（一个用于构建 Web 服务器的开源框架）中。

本指南旨在帮助你通过几个简单步骤，将 Better Auth 集成到你的 Nitro 应用中。

## 创建新的 Nitro 应用

首先使用以下命令来搭建一个新的 Nitro 应用：

```bash title="Terminal"
npx giget@latest nitro nitro-app --install
```

此命令将创建 `nitro-app` 目录并安装所有依赖。你现在可以在代码编辑器中打开 `nitro-app` 目录。

### Prisma 适配器设置

<Callout>
  本指南假设你对 Prisma 有基本的了解。如果你刚接触 Prisma，可以查看 [Prisma 文档](https://www.prisma.io/docs/getting-started)。

  本指南中使用的 `sqlite` 数据库不适用于生产环境。你应当将其替换为适合生产的数据库，如 `PostgreSQL`。
</Callout>

本指南中，我们将使用 Prisma 适配器。你可以通过以下命令安装 Prisma 客户端：

```package-install
@prisma/client
```

`prisma` 可以作为开发依赖安装，命令如下：

```package-install
-D prisma
```

通过运行以下命令生成 `prisma` 目录中的 `schema.prisma` 文件：

```bash title="Terminal"
npx prisma init
```

你现在可以用以下内容替换 `schema.prisma` 文件：

```prisma title="prisma/schema.prisma"
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

// 将被删除，仅用于生成 prisma 客户端
model Test {
  id   Int    @id @default(autoincrement())
  name String
}
```

确保在 `.env` 文件中更新 `DATABASE_URL`，指向你的数据库位置。

```txt title=".env"
DATABASE_URL="file:./dev.db"
```

运行以下命令生成 Prisma 客户端并同步数据库：

```bash title="Terminal"
npx prisma db push
```

### 安装及配置 Better Auth

按照 [安装指南](/docs/installation) 中的第 1 和第 2 步，在你的 Nitro 应用中安装 Better Auth 并设置环境变量。

完成后，在 `server/utils/auth.ts` 文件中创建你的 Better Auth 实例。

```ts title="server/utils/auth.ts"
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();
export const auth = betterAuth({
  database: prismaAdapter(prisma, { provider: "sqlite" }),
  emailAndPassword: { enabled: true },
});
```

### 更新 Prisma Schema

使用 Better Auth CLI 通过以下命令更新你的 Prisma schema，添加所需模型：

```bash title="Terminal"
npx auth generate --config server/utils/auth.ts
```

<Callout>
  `--config` 选项用于指定你创建 Better Auth 实例的文件路径。
</Callout>

然后前往 `prisma/schema.prisma` 文件并保存，以触发自动格式化。

保存后，你可以运行 `npx prisma db push` 命令更新数据库 schema。

## 挂载处理器

你现在可以在 Nitro 应用中挂载 Better Auth 处理器。方法是在 `server/routes/api/auth/[...all].ts` 文件中添加以下代码：

```ts title="server/routes/api/auth/[...all].ts"
export default defineEventHandler((event) => {
  return auth.handler(toWebRequest(event));
});
```
<Callout>
  这是一个 [catch-all](https://nitro.build/guide/routing#catch-all-route) 路由，将处理所有请求路径为 `/api/auth/*`。
</Callout>

### CORS

你可以通过创建插件为 Nitro 应用配置 CORS。

先安装 cors 包：

```package-install
cors
```

然后创建新文件 `server/plugins/cors.ts`，添加以下代码：

```ts title="server/plugins/cors.ts"
import cors from "cors";
export default defineNitroPlugin((plugin) => {
  plugin.h3App.use(
    fromNodeMiddleware(
      cors({
        origin: "*",
      }),
    ),
  );
});
```
<Callout>
  这将为所有路由启用 CORS。你可以自定义 `origin` 属性以允许特定域的请求。确保该配置与你的前端应用相匹配。
</Callout>

### 认证守卫/中间件

你可以在 Nitro 应用中添加认证守卫以保护需要认证的路由。方法是创建新文件 `server/utils/require-auth.ts`，并添加以下代码：

```ts title="server/utils/require-auth.ts"
import { EventHandler, H3Event } from "h3";
import { fromNodeHeaders } from "better-auth/node";

/**
 * 用于要求路由认证的中间件。
 *
 * 可扩展以检查特定角色或权限。
 */
export const requireAuth: EventHandler = async (event: H3Event) => {
  const headers = event.headers;

  const session = await auth.api.getSession({
    headers: headers,
  });
  if (!session)
    throw createError({
      statusCode: 401,
      statusMessage: "未授权",
    });
  // 你可以将 session 保存到 event 上下文以供后续使用
  event.context.auth = session;
};
```

你现在可以在路由中使用此事件处理器/中间件来保护路由：

```ts title="server/routes/api/secret.get.ts"
// 路由处理器的对象语法
export default defineEventHandler({
  // 用户必须登录才能访问此路由
  onRequest: [requireAuth],
  handler: async (event) => {
    setResponseStatus(event, 201, "机密数据");
    return { message: "机密数据" };
  },
});
```
