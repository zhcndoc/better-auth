title: Hono 集成
description: 将 Better Auth 集成到 Hono 中。
---

在开始之前，请确保你已经配置了 Better Auth 实例。如果还没有，请查看[安装指南](/docs/installation)。

### 挂载处理器

我们需要将处理器挂载到 Hono 的端点。

```ts
import { Hono } from "hono";
import { auth } from "./auth";
import { serve } from "@hono/node-server";

const app = new Hono();

app.on(["POST", "GET"], "/api/auth/*", (c) => {
	return auth.handler(c.req.raw);
});

serve(app);
```

### 跨域资源共享 (Cors)

要配置 CORS，你需要使用 `hono/cors` 提供的 `cors` 插件。

```ts
import { Hono } from "hono";
import { auth } from "./auth";
import { serve } from "@hono/node-server";
import { cors } from "hono/cors";
 
const app = new Hono();

app.use(
	"/api/auth/*", // 或替换为 "*" 以启用所有路由的 cors
	cors({
		origin: "http://localhost:3001", // 替换为你的源地址
		allowHeaders: ["Content-Type", "Authorization"],
		allowMethods: ["POST", "GET", "OPTIONS"],
		exposeHeaders: ["Content-Length"],
		maxAge: 600,
		credentials: true,
	}),
);

app.on(["POST", "GET"], "/api/auth/*", (c) => {
	return auth.handler(c.req.raw);
});

serve(app);
```

> **重要提示：** CORS 中间件必须在路由之前注册。这确保跨域请求在到达认证端点前能够被正确处理。

### 中间件

你可以添加一个中间件，用于将 `session` 和 `user` 保存在 `context` 中，并且为每个路由添加验证。

```ts
import { Hono } from "hono";
import { auth } from "./auth";
import { serve } from "@hono/node-server";
import { cors } from "hono/cors";
 
const app = new Hono<{
	Variables: {
		user: typeof auth.$Infer.Session.user | null;
		session: typeof auth.$Infer.Session.session | null;
	}
}>();

app.use("*", async (c, next) => {
	const session = await auth.api.getSession({ headers: c.req.raw.headers });

  	if (!session) {
    	c.set("user", null);
    	c.set("session", null);
    	await next();
        return;
  	}

  	c.set("user", session.user);
  	c.set("session", session.session);
  	await next();
});

app.on(["POST", "GET"], "/api/auth/*", (c) => {
	return auth.handler(c.req.raw);
});

serve(app);
```

这样你就可以在所有路由中访问 `user` 和 `session` 对象了。

```ts
app.get("/session", (c) => {
	const session = c.get("session");
	const user = c.get("user");
	
	if (!user) return c.body(null, 401);

  	return c.json({
	  session,
	  user,
	});
});
```

### 跨域 Cookie

默认情况下，所有 Better Auth 的 Cookie 都设置为 `SameSite=Lax`。如果你需要跨不同域使用 Cookie，则需要将 `SameSite` 设置为 `None` 且 `Secure=true`。不过我们建议尽可能使用子域名，这样就可以保持 `SameSite=Lax`。要启用跨子域 Cookie，只需在你的认证配置中开启 `crossSubDomainCookies`。

```ts title="auth.ts"
export const auth = createAuth({
  advanced: {
    crossSubDomainCookies: {
      enabled: true
    }
  }
})
```

如果你仍然需要设置 `SameSite=None` 和 `Secure=true`，可以通过 `createAuth` 配置里的 `cookieOptions` 全局调整这些属性。

```ts title="auth.ts"
export const auth = createAuth({
  advanced: {
    defaultCookieAttributes: {
      sameSite: "none",
      secure: true,
      partitioned: true // 新浏览器标准会强制要求此项用于跨域 Cookie
    }
  }
})
```

你也可以在 `cookies` 中单独为各个 Cookie 自定义属性。

```ts title="auth.ts"
export const auth = createAuth({
  advanced: {
    cookies: {
      sessionToken: {
        attributes: {
          sameSite: "none",
          secure: true,
          partitioned: true // 新浏览器标准会强制要求此项用于跨域 Cookie
        }
      }
    }
  }
})
```

### 客户端配置

当使用 Hono 客户端（`@hono/client`）请求受 Better Auth 保护的端点时，需要配置其发送带有凭证（Cookies）的跨域请求。

```ts title="api.ts"
import { hc } from "hono/client";
import type { AppType } from "./server"; // 你的 Hono 应用类型

const client = hc<AppType>("http://localhost:8787/", {
  init: {
    credentials: "include", // 发送跨域 Cookie 必需
  },
});

// 现在你的客户端请求将包含凭证
const response = await client.someProtectedEndpoint.$get();
```

此配置适用于以下情况：
- 开发时客户端和服务器处于不同域或端口
- 生产中执行跨域请求
- 需要随请求发送认证 Cookie

`credentials: "include"` 选项告诉 fetch 客户端即使是跨域请求也要发送 Cookie。这需要服务器端的 CORS 配置允许凭证，即设置了 `credentials: true`。

> **注意：** 确保服务器端的 CORS 配置中的 `origin` 值匹配客户端域名，并且在服务器和客户端的配置中都设定了 `credentials: true`。
